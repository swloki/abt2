# Excel 数据导入方案对比总结

## 背景

分析两种 Excel 数据导入方式的优劣：
- **方案A**：前端上传文件 → Astro 转发 → Rust 解析 → 数据库
- **方案B**：前端上传文件 → Astro 解析 → Rust 插入 → 数据库

## 当前架构

```
┌─────────┐      ┌─────────┐      ┌─────────┐      ┌──────────┐
│  前端   │ ──── │  Astro  │ ──── │  Rust   │ ──── │ 数据库   │
│ Astro   │      │  API    │      │ gRPC    │      │ PostgreSQL│
└─────────┘      └─────────┘      └─────────┘      └──────────┘
```

## 方案对比

| 维度 | 方案A (Rust解析) | 方案B (Astro解析) |
|------|-----------------|-------------------|
| **解析性能** | ✅ Rust calamine 极快 (~2s/10万行) | ❌ Node.js SheetJS 较慢 (~8s/10万行) |
| **内存效率** | ✅ 原生内存管理 | ❌ V8 堆内存，大文件易 OOM |
| **Astro→Rust 传输** | ✅ 文件 ~5-10MB | ❌ 数据 JSON ~30-50MB |
| **错误处理** | ✅ 统一在 Rust 层 | ❌ 需要两层协调 |
| **代码维护** | ✅ 逻辑集中 | ❌ 解析逻辑分散 |

## 结论

**保持方案A（当前方案）**，理由：

1. **Rust 解析更快** - calamine 性能远超 Node.js
2. **传输量更小** - 文件比解析后的数据小 3-5 倍
3. **逻辑集中** - 数据验证和解析都在一层，易维护

## 可优化点

当前 Astro 上传实现先完整读入内存再转发，可优化为流式转发：

```typescript
// 当前：先完整读入内存
const arrayBuffer = await file.arrayBuffer();

// 优化：流式转发（需要改动）
// 参考 docs/streaming-upload-optimization.md
```

但这对 10MB 以内的文件影响不大，优先级较低。
