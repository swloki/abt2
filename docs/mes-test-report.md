## 16. 建议修复优先级

1. **最高优先级**：修复 3 个 500 错误（批次/报检/入库列表页），这些是核心业务页面的入口
2. **高优先级**：修复"添加计划行"JS 失效问题，这是创建计划的核心功能
3. **高优先级**：修复取消工单后详情页 404 问题
4. **中优先级**：补全 UI 缺失字段（不良原因选择、备注输入等）
5. **低优先级**：实现 stub 页面的数据查询、未实现模块（排程看板等）
