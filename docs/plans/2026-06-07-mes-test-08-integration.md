# MES-08 端到端流程 + 跨模块集成 + 边界条件

## 1. 端到端生产流程测试（MTO 标准流程）

### 1.1 完整流程路径

```
创建 MTO 计划 → 确认计划 → 下达计划(生成工单) → 下达工单
→ 批次报工(多工序) → 创建检验 → 记录检验结果
→ 创建入库单 → 确认入库(触发倒冲) → 关闭工单
```

### 1.2 流程步骤

| 步骤 | ID | 操作 | URL | 预期结果 |
|------|-----|------|-----|---------|
| 1 | E2E-01 | 创建 MTO 生产计划 | POST /admin/mes/plans/create | 成功，跳转列表，列表出现新计划 |
| 2 | E2E-02 | 查看计划详情 | GET /admin/mes/plans/{plan_id} | 状态=草稿，显示明细行 |
| 3 | E2E-03 | 确认计划 | POST /admin/mes/plans/{id}/confirm | 状态=已确认 |
| 4 | E2E-04 | 下达计划 | POST /admin/mes/plans/{id}/release | 状态=进行中 |
| 5 | E2E-05 | 验证工单生成 | GET /admin/mes/orders | 工单数量=计划行数，状态=Planned |
| 6 | E2E-06 | 查看工单详情 | GET /admin/mes/orders/{id} | 显示关联计划信息 |
| 7 | E2E-07 | 下达工单 | POST /admin/mes/orders/{id}/release | 状态=已下达 |
| 8 | E2E-08 | 查看批次 | GET /admin/mes/batches | 验证是否有自动创建的批次 |
| 9 | E2E-09 | 批次报工（第1道工序） | POST /admin/mes/batches/{id}/confirm-step | 报工成功，current_step+1 |
| 10 | E2E-10 | 批次报工（后续工序） | POST /admin/mes/batches/{id}/confirm-step | 继续，直到最后工序 |
| 11 | E2E-11 | 创建检验 | POST /admin/mes/inspections/create | 检验记录创建 |
| 12 | E2E-12 | 记录检验结果(合格) | POST /admin/mes/inspections/{id}/record-result | 结果=合格 |
| 13 | E2E-13 | 创建入库单 | POST /admin/mes/receipts/create | 入库单创建，状态=草稿 |
| 14 | E2E-14 | 确认入库 | POST /admin/mes/receipts/{id}/confirm | 状态=已确认，倒冲触发 |
| 15 | E2E-15 | 关闭工单 | POST /admin/mes/orders/{id}/close | 工单关闭 |

### 1.3 验证检查点

| ID | 检查点 | 验证方式 |
|----|--------|---------|
| E2E-20 | 计划下达后工单数量正确 | 工单列表对比计划行数 |
| E2E-21 | 工单关联了正确的计划行 | 工单 plan_item_id 不为空 |
| E2E-22 | 报工后批次 completed_qty 更新 | 批次详情查看数量 |
| E2E-23 | 报工后 current_step 推进 | 批次详情查看工序 |
| E2E-24 | 检验结果与工单关联 | 检验 work_order_id 正确 |
| E2E-25 | 入库确认后 WMS 库存增加 | 查看库存查询 |
| E2E-26 | 入库确认触发倒冲 | backflush_triggered=true |
| E2E-27 | 倒冲扣减原材料 | 查看原材料库存 |
| E2E-28 | 工单版本号递增 | 每次操作后 version+1 |

## 4. 边界条件测试

### 4.1 输入验证

| ID | 测试项 | 操作 | 预期结果 |
|----|--------|------|---------|
| EDGE-01 | 空计划（无明细） | 创建计划不添加行 | 成功，详情显示"暂无计划明细" |
| EDGE-02 | 过去日期 | plan_date = 昨天 | 是否允许 |
| EDGE-03 | 开始 > 结束 | scheduled_start > scheduled_end | 是否允许 |
| EDGE-04 | 数量为 0 | completed_qty=0 | 是否允许 |
| EDGE-05 | 数量为负 | completed_qty=-1 | 被拒绝 |
| EDGE-06 | 数量超过计划 | completed_qty > planned_qty | 被拒绝或警告 |
| EDGE-07 | 检验样本为 0 | sample_qty=0 | 是否允许 |
| EDGE-08 | 入库数量为 0 | received_qty=0 | 是否允许 |
| EDGE-09 | 超长备注 | remark=10000 字符 | 截断或报错 |
| EDGE-10 | XSS 防护 | remark=`<script>alert(1)</script>` | 不执行 |
