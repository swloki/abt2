## 关键问题分析

### P1-#7: 库位列表筛选反序列化失败（根因）
`BinQueryParams.warehouse_id: Option<i64>` 缺少 `#[serde(default, deserialize_with = "empty_as_none")]`，导致 HTML 表单中空 `warehouse_id=""` 传到后端时 serde 无法将空字符串解析为 `i64`。此 bug 同时导致状态下拉筛选（#7）和文本搜索（#6，可能也是同样原因）失败。

**修复方案**：在 `BinQueryParams.warehouse_id` 字段添加 `#[serde(default, deserialize_with = "empty_as_none")]`。

### P1-#27: 库存列表库区联动消失（HTMX 竞态）
仓库下拉的 `change` 事件同时触发 form 的 table 请求和下拉自身的 zones 请求。两个并发 HTMX 请求响应顺序不确定，导致 zones 响应到达时目标元素已不在 DOM 中。

**修复方案**：将 zones 请求改为在 table 请求成功后触发（使用 `htmx:afterRequest` 事件），或让 table 端点在响应中包含更新的 zone-select HTML。

### P2-#5: 容量上限格式
列表页 `capacity_limit` 直接输出 Decimal 未格式化，显示 `800.000000`。

**修复方案**：使用格式化输出如 `{:.2}` 或自定义格式化函数。
