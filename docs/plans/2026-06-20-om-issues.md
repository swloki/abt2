## 修复结果（阶段 B 修复 + 阶段 C 回归验证）

| # | 问题 | 修复方案 | 回归 |
|---|------|---------|------|
| 0 | P0 状态机全失败 | 新增 migration `059_outsourcing_state_transitions.sql`（8 态 + 7 转换规则）+ `create` 加 `transition(...,"Draft",None)` | ✅ 单9 Draft→Sent 转换成功，state_logs 完整 |
| 1 | 虚拟仓库下拉空 | migration 059 `UPDATE warehouses SET is_virtual=true WHERE name='委外仓'` | ✅ 下拉显示"委外仓" |
| 1b | 可选字段空串反序列化 | `CreateForm` 的 `work_order_id`/`routing_id` 加 `#[serde(deserialize_with="empty_as_none")]` | ✅ 不关联工单/工序可创建（单9） |
| 1c | 缺发送按钮 + 无状态门控 | 详情页新增"发料"主按钮（Draft）+ 按 `OutsourcingStatus` 门控各操作按钮 + 新增 send modal | ✅ Draft 显示 发料/记录节点/转自制/取消，收货登记隐藏 |
| 4 | 关联工单显示原始 ID | `detail_page` 加载 `work_order_name`（`WorkOrderService::find_by_id`）显示工单号 | ✅ 显示 WO-2026-06-000790 |
| 2 | 供应商筛选是 ID 文本框 | 列表筛选栏供应商改为 select（列供应商名称，value=supplier_id） | ✅ select：全部供应商 + 2 供应商 |
| 3 | 日期初始显示 0 | **非 bug**：原生 `<input type=date>` 的无障碍树渲染为 spinbutton，实际 UI 显示正常日期占位符 | ⚪️ 无需修复 |
| 5 | 类型命名不一致 | 列表 `type_label` "产品全委外" → "整体委外"（与表单/详情统一） | ✅ |
| 6 | send 发料调拨源仓库=0 | 已实现：委外单加 `source_warehouse_id` 字段（migration 060 + model/repo + 创建表单加"发料源仓库"必填 select + send 用 `order.source_warehouse_id` 作调拨 `from_warehouse`，设计文档 05 已同步） | ✅ 单11 send 完整成功（调拨单 #49 from=23327→23322 dispatched，sent_qty=100，SendMaterial 节点记录） |

**核心结论**：P0 状态机已修复，委外单可正常创建并发料流转（Draft→Sent）。剩余 #6（发料源仓库）是设计层面缺失，需确认"委外发料从哪个仓库扣减库存"后才能让 send 的库存调拨真正生效。
