字段当前处置依据
行号明细列删除Odoo 用隐藏 sequence 拖拽排序；ERPNext/OFBiz 均无行号列
操作员头部只读删除三家均由系统按当前用户自动记录（Odoo user_id / ERPNext owner / OFBiz issuedByUserLoginId）
仓库行内 cell挪 header三家均把仓库放单据头（Odoo location_id / ERPNext from_warehouse / OFBiz facilityId），行内仅在多库位组时可选
需求量(BOM)明细列合并小字工单模式留作只读上下文（Odoo "Demand" 同行只读）；手动模式隐藏
已领明细列合并小字Odoo 与本次合并为累计 quantity；这里做只读小字，保留现场对账价值
可用量明细列合并小字三家均不占正经列（Odoo widget / ERPNext 折叠 / OFBiz 用缺口替代）；缺料时红字提示
单位明细列并入产品格ERPNext 仅 UOM 不同时显示；OFBiz 存 Product 不显示
产品明细列保留三家必留
库位行内 cell保留行内同物料可在多库位领取；保留行内窄列（可空）
批次明细列保留行内批次物料必填；三家均保留（Odoo/ERPNext 默认折叠，按 tracking 显示）
本次领料明细列保留唯一可编辑数量，三家核心
关联工单头部保留ABT 工单领料链路必需；空=手动模式
领料日期头部保留后端 requisition_date 必填
