## 问题

ABT 后端零鉴权，所有 gRPC 接口裸奔。User/Role/Permission 表存在但未使用，operator_id 全部传 None。密码"哈希"是 `format!("hashed:{}", password)`，等于明文。

## 最终方案

用户名密码登录 + JWT + handler 手动权限检查 + 审计日志。

## 架构决策记录

| 决策 | 选项 | 理由 |
|------|------|------|
| 登录方式 | 用户名密码（非微信） | 微信登录以后加，只是一个新 RPC |
| 密码哈希 | argon2 | 现有 format!("hashed:{}", pw) 等于明文，必须换 |
| JWT 验证位置 | tonic sync interceptor | 纯 CPU 操作，不需要 DB |
| 权限检查位置 | handler 手动调用 check_permission() | 显式声明，不会出隐蔽 bug |
| 权限数据来源 | JWT claims（内存匹配） | 不查 DB，纯内存操作 |
| 资源定义位置 | 代码（ResourceProvider trait） | 代码是 source of truth，带中文名和描述 |
| 资源同步 | 启动时自动收集，可选同步到 DB | 管理页面可从内存 API 读 |
| 数据库资源表 | 废弃 resources/permissions/actions 表 | 只保留 role_permissions |
| TLS | nginx 反代处理 | gRPC 服务本身不加密 |

## 核心流程

```
登录:

1. 前端 -> gRPC Login(username, password)
2. 后端 argon2 验证密码
3. 查询用户角色和权限 (role_permissions 表)
4. 签发 JWT (包含 user_id, is_super_admin, permissions: ["product:read", ...])
5. 返回 JWT

每次请求:

1. 前端带 Authorization: Bearer <jwt>
2. interceptor (同步): 验证 JWT 签名+过期, 提取 claims 放入 extensions
   - AuthService.Login 跳过验证
3. handler 开头: auth.check_permission("product", "write")?
   - 匹配 JWT claims 中的 "product:write"
   - super_admin 跳过检查
4. handler 业务逻辑, operator_id = auth.user_id
5. 审计: 复用已有 PermissionRepo::insert_audit_log
```

## AuthContext 和 check_permission

```rust
/// 从 gRPC request extensions 中提取的认证上下文
pub struct AuthContext {
    pub user_id: i64,
    pub username: String,
    pub is_super_admin: bool,
    pub permissions: Vec<String>,  // ["product:read", "product:write", ...]
}

impl AuthContext {
    /// 从 gRPC request 的 extensions 中提取
    pub fn from_request<T>(request: &Request<T>) -> Result<Self, Status> {
        request.extensions()
            .get::<AuthContext>()
            .cloned()
            .ok_or_else(|| Status::internal("AuthContext not found in extensions"))
    }

    /// 检查权限。super_admin 自动通过。
    pub fn check_permission(&self, resource: &str, action: &str) -> Result<(), Status> {
        if self.is_super_admin {
            return Ok(());
        }
        let required = format!("{}:{}", resource, action);
        if self.permissions.contains(&required) {
            Ok(())
        } else {
            Err(Status::permission_denied(
                format!("No permission for {}:{}", resource, action)
            ))
        }
    }
}
```

## Interceptor 实现

```rust
// abt-grpc/src/interceptors/auth.rs

use tonic::{Request, Status};

/// JWT 验证 interceptor (同步)
/// - 提取 Authorization: Bearer <jwt>
/// - 验证签名和过期
/// - 提取 claims 构建 AuthContext
/// - 放入 request extensions
/// - AuthService 的 Login 跳过
pub fn auth_interceptor(request: Request<()>) -> Result<Request<()>, Status> {
    // 1. 提取 Authorization header
    let token = request.metadata()
        .get("authorization")
        .and_then(|v| v.to_str().ok())
        .and_then(|v| v.strip_prefix("Bearer "));

    // 2. AuthService.Login 不需要 token (通过 method path 判断)
    // tonic interceptor 拿不到 method path，需要用 with_interceptor 只对需要的服务加

    let token = match token {
        Some(t) => t,
        None => return Err(Status::unauthenticated("Missing authorization token")),
    };

    // 3. 验证 JWT
    let claims = decode_jwt(token)?;  // jsonwebtoken::decode

    // 4. 构建 AuthContext 放入 extensions
    let auth_ctx = AuthContext {
        user_id: claims.sub,
        username: claims.username,
        is_super_admin: claims.is_super_admin,
        permissions: claims.permissions,
    };

    let mut request = request;
    request.extensions_mut().insert(auth_ctx);
    Ok(request)
}
```

## 测试清单

```
AuthService::login
  [ ] 正确用户名+密码 -> 返回 JWT
  [ ] 错误密码 -> Unauthenticated
  [ ] 用户不存在 -> Unauthenticated
  [ ] is_active=false -> 拒绝登录
  [ ] JWT 包含正确的 permissions

AuthInterceptor
  [ ] 有效 JWT -> 通过
  [ ] 无 JWT -> Unauthenticated
  [ ] 过期 JWT -> Unauthenticated
  [ ] 错误签名 -> Unauthenticated
  [ ] AuthService.Login 跳过验证

check_permission
  [ ] 有权限 -> 通过
  [ ] 无权限 -> PermissionDenied
  [ ] super_admin -> 全部通过

argon2 密码
  [ ] 创建用户时用 argon2 哈希
  [ ] 登录时用 argon2 验证
```

## Success Criteria

1. 用户能通过用户名密码登录，获取 JWT
2. 所有 gRPC 接口需要有效 JWT 才能调用（AuthService 除外）
3. 非超级管理员的用户，根据 role_permissions 检查权限
4. 所有写操作记录审计日志，包含 operator_id
5. 密码使用 argon2 哈希存储
6. 管理页面能通过 API 看到带中文名的资源列表
