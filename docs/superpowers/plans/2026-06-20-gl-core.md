# GL 内核 Implementation Plan（Plan A · 财务 roadmap 第一期）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 搭建总账内核——科目表、复式记账凭证、会计期间、试算平衡，让后续发票/业财过账（Plan B/C）有地方落账。

**Architecture:** 独立 `gl` 域（与 fms 平级），含 `account`（科目表）/`entry`（凭证）/`period`（期间）三子模块。凭证走业财一体：业务单据 posted 同事务调 `GlEntryService::post_from_source` 生成 `gl_entries`+`gl_entry_lines`（本 Plan A 只实现手工凭证 `create_manual` + 过账校验，post_from_source 的业务衔接留 Plan B/C）。试算平衡按期间汇总科目借贷。期间锁定在凭证 post 时校验。

**Tech Stack:** Rust 2024 / sqlx / async-trait / PostgreSQL / chrono / rust_decimal

**Spec:** `docs/superpowers/specs/2026-06-20-gl-invoice-design.md`（第 1-5、7、8、11 节为本 Plan 范围；第 6 节发票、第 9 节业务衔接、第 10 节前端属 Plan B/C/D）

**A1 migration（056）表结构增补**（在原表定义上加，不是另起一张表）：
- `gl_accounts`：**移除 `status`**；新增 `reconcile BOOLEAN NOT NULL DEFAULT FALSE`、`disabled BOOLEAN NOT NULL DEFAULT FALSE`、`opening_balance DECIMAL(18,6) NOT NULL DEFAULT 0`、`currency VARCHAR(10) NOT NULL DEFAULT 'CNY'`

**A2 科目模块增补**：
- `gl/enums.rs`：`EntryStatus` 加 `Cancelled=3 => "Cancelled"`；**移除 `AccountStatus`**（用 `disabled: bool` 取代）
- `GlAccount` model：**移除 `status: AccountStatus`**，加 `reconcile: bool, disabled: bool, opening_balance: Decimal, currency: String`
- `CreateGlAccountReq` 加 `reconcile: bool, opening_balance: Decimal, currency: String`；`UpdateGlAccountReq` 用 `disabled: Option<bool>` 替代 `status`
- Repo `list` 的过滤条件：原 `status` → `disabled`

**A4 凭证模块增补**：
- `GlEntry` model 加 `voucher_type: String, is_opening: bool`
- `GlEntryLine` model 加 `amount_currency: Decimal, currency: String, exchange_rate: Decimal, project_id: Option<i64>`
- `GlEntryLineInput` 同步加 `project_id: Option<i64>`（amount_currency/currency/exchange_rate 本期默认 0/CNY/1，create 时不强制录入）
- `CreateManualEntryReq` 加 `voucher_type: String, is_opening: bool`
- `GlEntryService` trait **新增方法**：
  - `async fn cancel(&self, ctx: &ServiceContext, db: PgExecutor<'_>, id: i64) -> Result<()>` — posted→cancelled，记审计；余额查询自动排除 cancelled 凭证（无需红字反向分录）
  - `async fn general_ledger(&self, ctx: &ServiceContext, db: PgExecutor<'_>, account_id: i64, from: Option<NaiveDate>, to: Option<NaiveDate>) -> Result<Vec<GlDetailRow>>`
  - `get_account_balance` 签名补 `as_of_date: Option<NaiveDate>` 参数（按日期切片）
- 新增 `GlDetailRow { entry_id: i64, doc_number: String, entry_date: NaiveDate, memo: String, counterpart_account_id: Option<i64>, debit: Decimal, credit: Decimal, running_balance: Decimal }`
- implt：
  - `cancel`：取 entry 校验 status==Posted → `update_status(id, Cancelled, version)` 乐观锁（rows==0 ConcurrentConflict）→ audit（Transition posted→cancelled）
  - `general_ledger`：查该科目 posted 且未删的分录行（JOIN gl_entries.status=2），按 entry_date 排序，`counterpart` 取同凭证其他行第一行的 account_id，`running_balance` 按 balance_direction 累加
  - `get_account_balance`：`account.opening_balance + Σ(该科目 posted 分录 debit/credit，按 balance_direction 定向：借方科目=+debit-credit，贷方科目=+credit-debit)`，可选 `as_of_date` 过滤 `entry_date`
  - `trial_balance` 的余额口径同步：只统计 status=posted 凭证；含 opening_balance

**A5 e2e 增补用例**（k1-k3 之后追加）：
- `k4_entry_cancel_excludes_from_balance`：post 一张凭证记下科目余额 → cancel → 验证 entry.status=Cancelled + get_account_balance 已减去该凭证金额
- `k5_general_ledger_running_balance`：对某科目调 general_ledger → 验证返回分录流水按日期排序、running_balance 正确累加
- `k6_opening_balance`：建科目时设 opening_balance=500 → get_account_balance 返回 ≥ 500（含期初）

**本期不做（spec 第 13 节留后续）**：unpost 回 draft、自动红字反向凭证、自动损益结转（M4）、可配置维度系统、物化余额表、多账簿/多公司、自动汇兑损益重估（M8）。

-- account_type: 1资产/2负债/3权益/4收入/5成本/6费用
-- balance_direction: 1借/2贷
-- entry status: 1draft/2posted
-- period status: 1open/2closed

CREATE TABLE gl_accounts (
    id                BIGSERIAL    PRIMARY KEY,
    code              VARCHAR(40)  NOT NULL UNIQUE,
    name              VARCHAR(100) NOT NULL,
    account_type      SMALLINT     NOT NULL,        -- 1..6
    parent_id         BIGINT       REFERENCES gl_accounts(id),
    is_detail         BOOLEAN      NOT NULL DEFAULT TRUE,
    balance_direction SMALLINT     NOT NULL,        -- 1借/2贷
    company_id        BIGINT       NOT NULL DEFAULT 1,
    status            SMALLINT     NOT NULL DEFAULT 1, -- 1活跃/2停用
    version           INTEGER      NOT NULL DEFAULT 1,
    created_at        TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at        TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    deleted_at        TIMESTAMPTZ
);
CREATE INDEX idx_gl_accounts_parent ON gl_accounts(parent_id);
COMMENT ON TABLE gl_accounts IS '科目表 — 只有 is_detail=TRUE 的末级科目可被凭证引用';

CREATE TABLE accounting_periods (
    id           BIGSERIAL   PRIMARY KEY,
    name         VARCHAR(20) NOT NULL UNIQUE,       -- 2026-06
    start_date   DATE        NOT NULL,
    end_date     DATE        NOT NULL,
    status       SMALLINT    NOT NULL DEFAULT 1,    -- 1open/2closed

    closed_by    BIGINT,

    updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE gl_entries (

    doc_number    VARCHAR(40) NOT NULL,

    entry_date    DATE        NOT NULL,

    source_id     BIGINT      NOT NULL DEFAULT 0,
    description   VARCHAR(300) NOT NULL DEFAULT '',
    status        SMALLINT    NOT NULL DEFAULT 1,   -- 1draft/2posted
    total_debit   DECIMAL(18,6) NOT NULL DEFAULT 0,
    total_credit  DECIMAL(18,6) NOT NULL DEFAULT 0,
    operator_id   BIGINT      NOT NULL,

);

CREATE INDEX idx_gl_entries_period ON gl_entries(period);

    entry_id      BIGINT      NOT NULL REFERENCES gl_entries(id),
    account_id    BIGINT      NOT NULL REFERENCES gl_accounts(id),
    debit         DECIMAL(18,6) NOT NULL DEFAULT 0,
    credit        DECIMAL(18,6) NOT NULL DEFAULT 0,
    cost_center   BIGINT,
    profit_center BIGINT,
    memo          VARCHAR(200) NOT NULL DEFAULT ''
);
CREATE INDEX idx_gl_entry_lines_entry   ON gl_entry_lines(entry_id);
CREATE INDEX idx_gl_entry_lines_account ON gl_entry_lines(account_id);

CREATE TABLE gl_account_mappings (
    id          BIGSERIAL   PRIMARY KEY,
    mapping_key VARCHAR(40) NOT NULL,    -- default_ar/default_ap/default_revenue/default_inventory/default_tax_output/default_tax_input/default_bank/default_expense

);

    ('1122',    '应收账款',         1, NULL, TRUE, 1),
    ('2202',    '应付账款',         2, NULL, TRUE, 2),
    ('2221.01', '应交税费-销项税',  2, NULL, TRUE, 2),
    ('2221.02', '应交税费-进项税',  2, NULL, TRUE, 2),
    ('4001',    '实收资本',         3, NULL, TRUE, 2),
    ('5001',    '主营业务收入',     4, NULL, TRUE, 2),
    ('1405',    '库存商品',         1, NULL, TRUE, 1),

    ('4103',    '本年利润',         3, NULL, TRUE, 2)
ON CONFLICT (code) DO NOTHING;

-- ── 默认科目映射（指向上面 seed 的科目，按 code 反查 id）──
INSERT INTO gl_account_mappings (mapping_key, account_id, product_id)

    ('default_ar'), ('default_ap'), ('default_revenue'), ('default_inventory'),

    WHEN 'default_inventory'   THEN '1405'

    WHEN 'default_tax_input'   THEN '2221.02'
    WHEN 'default_bank'        THEN '1002'

SELECT to_char(d, 'YYYY-MM'), d, (d + INTERVAL '1 month - 1 day')::date, 1, '2026'
FROM generate_series('2026-01-01'::date, '2026-06-01'::date, INTERVAL '1 month') AS d

ON CONFLICT DO NOTHING;

// AccountType: 1资产/2负债/3权益/4收入/5成本/6费用
// BalanceDirection: 1借/2贷
// （均 #[repr(i16)]，参照 fms/enums.rs 的 impl_sqlx_traits!/impl_serde_traits! 宏模式实现 sqlx+serde）

macro_rules! def_enum {
    ($name:ident { $($v:ident = $n:expr => $s:literal),+ $(,)? }) => {
        #[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
        #[repr(i16)]
        pub enum $name { $($v = $n),+ }
        impl $name {
            pub fn from_i16(v: i16) -> Option<Self> { Some(match v { $($n => Self::$v,)+ _ => return None }) }
            pub fn as_i16(self) -> i16 { self as i16 }
            pub fn as_str(self) -> &'static str { match self { $(Self::$v => $s),+ } }
        }
        impl sqlx::Type<sqlx::Postgres> for $name {
            fn type_info() -> sqlx::postgres::PgTypeInfo { <i16 as sqlx::Type<sqlx::Postgres>>::type_info() }
            fn encode_by_ref(&self, b: &mut sqlx::postgres::PgArgumentBuffer)
                -> Result<sqlx::encode::IsNull, Box<dyn std::error::Error + Send + Sync>> {
                <i16 as sqlx::Encode<'_, sqlx::Postgres>>::encode_by_ref(&self.as_i16(), b)
            }
        }
        impl sqlx::Decode<'_, sqlx::Postgres> for $name {
            fn decode(val: sqlx::postgres::PgValueRef<'_>) -> Result<Self, Box<dyn std::error::Error + Send + Sync>> {
                let v = <i16 as sqlx::Decode<'_, sqlx::Postgres>>::decode(val)?;
                Self::from_i16(v).ok_or_else(|| format!("unknown {}: {v}", stringify!($name)).into())
            }
        }
        impl Serialize for $name { fn serialize<S: serde::Serializer>(&self, s: S) -> Result<S::Ok, S::Error> { s.serialize_i16(self.as_i16()) } }
        impl<'de> Deserialize<'de> for $name { fn deserialize<D: serde::Deserializer<'de>>(d: D) -> Result<Self, D::Error> { let v = i16::deserialize(d)?; Self::from_i16(v).ok_or_else(|| serde::de::Error::custom(format!("unknown {}", stringify!($name)))) } }
    };
}

pub struct GlAccountRepo;
impl GlAccountRepo {
    pub async fn create(executor: PgExecutor<'_>, req: &CreateGlAccountReq) -> Result<i64> {

    pub async fn get_by_id(executor: PgExecutor<'_>, id: i64) -> Result<Option<GlAccount>> {
        let row = sqlx::query_as::<_, GlAccount>(sqlx::AssertSqlSafe(

    // 调整：close 前确保该期无 draft；用新期间测：close 一个空期间 → 在该期建凭证 create_manual 直接 resolve_open 失败

    // 选一个 open 期间并关闭

    let p = periods.iter().find(|p| p.status == abt_core::gl::enums::PeriodStatus::Open).unwrap();
    period_svc.close(&ctx, &mut conn, p.id).await.expect("close empty period");
    // 该期间内建凭证应 resolve_open 失败
    let bank = seed_account(&app, "BANK2", AccountType::Asset, BalanceDirection::Debit).await;
