- [ ] **Step 5: Implement in DepartmentServiceImpl**

Modify `abt/src/implt/department_service_impl.rs` — add implementations that delegate to `UserDepartmentRoleRepo`:

```rust
async fn assign_user_dept_roles(
    &self,
    _operator_id: Option<i64>,
    user_id: i64,
    assignments: Vec<DeptRole>,
    executor: Executor<'_>,
) -> Result<()> {
    // Validate: all departments exist and are active
    // Validate: all roles exist and are NOT system roles
    for assignment in &assignments {
        let role = RoleRepo::find_by_id_with_executor(executor.borrow(), assignment.role_id).await?
            .ok_or_else(|| anyhow!("Role {} not found", assignment.role_id))?;
        if role.is_system_role {
            return Err(anyhow!("Cannot assign system role {} to a department", assignment.role_id));
        }
    }
    UserDepartmentRoleRepo::assign(executor, user_id, &assignments).await
}
```

- [ ] **Step 7: Update auth handler**

Modify `abt-grpc/src/handlers/auth.rs` — add `switch_department` handler:

```rust
async fn switch_department(
    &self,
    request: Request<SwitchDepartmentRequest>,
) -> GrpcResult<LoginResponse> {
    let auth = extract_auth(&request)?;
    let req = request.into_inner();
    let (token, expires_at, _claims) = self.state()
        .auth_service()
        .switch_department(auth.user_id, req.department_id)
        .await
        .map_err(|e| Status::internal(e.to_string()))?;

    // Build response (reuse LoginResponse structure)
    Ok(Response::new(LoginResponse { token, expires_at, ... }))
}
```

- [ ] **Step 8: Update existing handlers that use department_id**

Any handler that operates on business resources now needs to accept `department_id` from the request. The `require_permission` macro reads it from `x-department-id` metadata, so handlers themselves don't need to extract it — the macro does it internally.

However, if any handler needs the department_id for business logic (not just permission checks), it should extract it from the request metadata or request body.
