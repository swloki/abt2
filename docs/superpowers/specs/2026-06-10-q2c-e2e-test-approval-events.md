# 审批流与事件通知机制

> **所属方案**：生产型 ERP 全链路 E2E 测试
> **本文档定义**：审批矩阵、跨模块事件链、通知联动机制

---

## 1. 审批流矩阵

### 1.1 审批节点总览

| 序号 | 审批节点 | 触发条件 | 审批类型 | 审批人 | 通过后动作 | 驳回后动作 |
|------|----------|----------|----------|--------|-----------|-----------|
| A1 | 报价审批 (S3) | 折扣率 > 15% 或金额 > 50万 | 或签 | 销售经理 → 总经理 | 报价生效，可转合同 | 退回销售修改 |
| A2 | 销售订单审批 (S5) | 信用额度超限 / 特殊条款 | 会签 | 销售主管 + 财务主管 | 触发 MRP 运算 | 退回修改或取消 |
| A3 | 采购审批 (PU3) | 金额 > 10万 / 非常规物料 | 逐级 | 采购经理 → 财务总监 | 生成 PO 发送供应商 | 退回采购修改 |
| A4 | 工单下达审批 (M1) | 紧急工单 / 超产能 | 单签 | 生产经理 | 工单下达，触发领料 | 工单挂起 |
| A5 | 超领审批 (M2-E1) | 领料超出 BOM 标准用量 | 单签 | 生产经理 | 允许超领出库 | 退回调整领料数量 |
| A6 | 报废审批 (M4-E3) | 报废数量 > 工单 5% | 会签 | 质量主管 + 生产经理 + 财务 | 报废入废品仓 + 成本重算 | 触发 MRB 评审 |
| A7 | 发货审批 (W3) | 超额发货 / 信用冻结客户 | 单签 | 销售经理 | 发货出库 + 触发应收 | 发货拦截 |
| A8 | 开票审批 (F3) | 红字发票 / 金额差异 > 5% | 逐级 | 财务主管 → 总经理 | 发票开出寄送客户 | 退回财务修正 |
| A9 | 合同变更审批 (S4-E1) | 价格调整/交期变更 | 逐级 | 销售经理 → 总经理 | 合同变更生效 | 退回修改 |
| A10 | PO 变更审批 (PU4-E1) | 数量/价格/交期变更 | 逐级 | 采购经理 → 财务总监 | PO 变更生效，通知供应商 | 退回修改 |
| A11 | MRB 评审 (M4-E1) | 质检不合格处置 | 会签 | 质量主管 + 生产经理 + 销售 | 处置决定生效（返工/让步/报废） | 继续评审或升级 |
| A12 | 应付确认审批 (FP1) | AP 金额 > 50万 / 首次合作供应商 | 逐级 | 财务主管 → 财务总监 | AP 确认，可发起付款 | 退回核实 |
| A13 | 付款审批 (FP3) | 付款金额 > 20万 | 单签 | 财务主管 | 执行付款 | 暂停付款 |

### 1.2 审批类型定义

| 审批类型 | 规则说明 | 测试关注点 |
|----------|----------|-----------|
| **单签** | 任一指定审批人通过即生效 | 1. 审批人通过后状态变更是否正确；2. 审批记录是否完整 |
| **或签** | 多人按顺序流转，任一通过即结束 | 1. 第一人通过后第二人不再收到待办；2. 第一人拒绝后是否流转给第二人 |
| **会签** | 所有审批人都需通过才算通过 | 1. 所有人通过才生效；2. 任一人拒绝即驳回；3. 部分通过时等待状态正确 |
| **逐级** | 按层级依次审批，上级否决则终止 | 1. 下级通过后才到上级；2. 下级拒绝不触发上级；3. 上级拒绝直接终止 |

### 1.3 审批流异常场景测试

| 场景编号 | 异常场景 | 预期行为 | 验证方法 |
|----------|----------|----------|----------|
| AP-E1 | **驳回重提**：审批人驳回后，提交人修改重新提交 | 新审批实例创建，引用原单据；旧审批记录保留 | 检查审批历史中包含驳回和重提记录 |
| AP-E2 | **审批超时升级**：超过 24h 未审批 | 自动升级到上级审批人；原审批人收到催办通知 | 等待超时后检查审批待办列表 |
| AP-E3 | **委托代理**：审批人请假，委托他人审批 | 代理人可在审批列表看到待办；审批记录标注代理信息 | 以代理人身份查看待办并审批 |
| AP-E4 | **提交人撤回**：审批前提交人主动撤回 | 审批流程终止；业务单据恢复到编辑状态 | 撤回后检查单据状态 |
| AP-E5 | **会签部分拒绝**：会签中一人拒绝但其他人未操作 | 整体标记为"驳回"；未操作的审批人待办清除 | 检查审批状态和待办列表 |
| AP-E6 | **并行审批竞态**：或签中两人同时审批 | 第一个提交的人生效；第二个收到"已审批"提示 | 模拟两人同时打开审批页面 |
| AP-E7 | **审批中单据被修改**：审批过程中业务数据被其他人修改 | 审批自动挂起，提示数据已变更需重新审批 | 修改后检查审批状态 |
| AP-E8 | **审批历史完整性**：多次驳回-重提的完整历史 | 所有审批动作按时间排列，含操作人、时间、意见 | 查看审批历史日志 |

## 2. 跨模块事件链

### 2.1 核心事件链

以下是 Quote-to-Cash 主线上按时间顺序的 **5 条核心事件链**，每条事件链定义了触发事件、下游动作和验证要点。

#### 事件链 EC-1：销售订单创建 → MRP 触发

```
[Sale Order 创建]
  │ Event: order_created
  ├→ 通知：计划员（站内信 + 企微）
  ├→ 动作：库存预留（Inventory Reservation）
  └→ 动作：触发 MRP 运算请求
       │ Event: mrp_triggered
       └→ 验证：MRP 是否正确接收 SO 明细
```

| 验证点 | 预期 |
|--------|------|
| SO 创建后库存预留 | SO 明细产品被锁定库存预留记录 |
| 计划员收到通知 | 站内信内容含 SO 单号、产品、数量 |
| MRP 自动触发 | MRP 运算日志中可见触发来源 SO |
| 重复触发幂等 | 同一 SO 不产生重复 MRP 任务 |

#### 事件链 EC-2：MRP 完成 → 采购 + 生产并行

```
[MRP 运算完成]
  │ Event: mrp_completed
  ├→ 动作：生成采购需求（外购件）
  │    │ Event: purchase_request_generated
  │    └→ 通知：采购专员（站内信）
  ├→ 动作：生成生产工单建议（自制件）
  │    │ Event: work_order_suggested
  │    └→ 通知：生产主管（站内信 + 企微）
  └→ 动作：更新 SO 需求满足率
       └→ 验证：SO 需求 100% 被覆盖
```

| 验证点 | 预期 |
|--------|------|
| 采购需求数量 | 与 MRP 净需求数量一致 |
| 工单建议数量 | 与 MRP 净需求数量一致 |
| 自制/外购分类 | BOM 中标记为自制的走工单，外购的走采购 |
| 并行触发 | 采购和生产通知几乎同时发出 |

#### 事件链 EC-3：采购到货 → 质检 → 入库 → 应付

```
[采购收货入库]
  │ Event: goods_received
  ├→ 动作：触发来料质检任务
  │    │ Event: inspection_task_created
  │    └→ 通知：质检员（站内信）
  ├→ 动作：库存增加（待检状态，非可用）
  └→ 动作：生成应付暂估
       │ [质检通过]
       │ Event: inspection_passed
       ├→ 动作：库存状态转为"可用"
       ├→ 动作：应付暂估转正式应付 (AP)
       └→ 动作：更新 PO 未交数量
            │ [质检不合格]
            │ Event: inspection_failed
            ├→ 动作：触发 MRB 评审
            ├→ 动作：库存转隔离仓
            └→ 通知：质量主管 + 采购 + 供应商（邮件）
```

| 验证点 | 预期 |
|--------|------|
| 收货后库存状态 | "待检"，不计入 ATP 可用库存 |
| 质检通过后 | 库存转为"可用"，ATP 更新 |
| 应付暂估 | 收货时生成暂估凭证，质检后转为正式 AP |
| 质检不合格 | 库存移入隔离仓，MRB 流程启动 |

#### 事件链 EC-4：生产完工 → 质检 → 入库 → 成本

```
[生产报工完成（末道工序）]
  │ Event: work_reported
  ├→ 动作：触发出成品质检任务
  │    └→ 通知：质检员（站内信）
  ├→ 动作：归集工时和用料到工单
  └→ 动作：触发成本核算
       │ [质检合格]
       │ Event: quality_passed
       ├→ 动作：成品入库 + 库存增加
       ├→ 动作：工单状态 → 完工
       ├→ 动作：成本核算完成（材料 + 人工 + 制造费用）
       └→ 通知：计划员 + 仓管（站内信）
            │ [质检不合格]
            │ Event: quality_failed
            ├→ 动作：触发 MRB
            ├→ 动作：不合格品入隔离仓
            └→ 动作：成本差异标记
```

| 验证点 | 预期 |
|--------|------|
| 报工后工时归集 | 工单实际工时 = 各工序工时之和 |
| 成品入库数量 | 与报工合格数量一致 |
| 成本核算准确性 | 材料成本 + 人工成本 + 制造费用 = 总成本 |
| 工单完工通知 | 计划员收到工单完工通知 |

#### 事件链 EC-5：发货 → 应收 → 开票 → 收款 → 核销

```
[发货出库]
  │ Event: shipment_out
  ├→ 动作：库存扣减
  ├→ 动作：生成应收凭证 (AR)
  └→ 通知：财务会计 + 客户（邮件）
       │ [客户签收]
       │ Event: customer_received
       ├→ 动作：确认收入
       ├→ 动作：触发开票流程
       └→ 通知：财务会计（站内信）
            │ [开票完成]
            │ Event: invoice_sent
            ├→ 动作：发票寄送客户
            └→ 通知：销售（站内信）
                 │ [收款到账]
                 │ Event: payment_received
                 ├→ 动作：收款凭证 + 银行入账
                 ├→ 动作：自动核销（AR 冲收款）
                 ├→ 动作：释放客户信用额度
                 └→ 动作：更新 SO 状态 → 已结算
                      │ Event: order_settled
                      └→ 流程结束
```

| 验证点 | 预期 |
|--------|------|
| 发货后 AR 自动生成 | AR 金额 = 发货数量 × 单价 × (1 + 税率) |
| 签收后收入确认 | 收入确认时点为客户签收日 |
| 开票金额与 AR 一致 | 发票金额 = AR 金额（含税） |
| 收款后自动核销 | 核销金额 = 收款金额；AR 余额减少 |
| 信用额度释放 | 核销后客户信用额度恢复 |
| SO 状态流转 | SO：待发货 → 已发货 → 已签收 → 已开票 → 已收款 → 已结算 |

## 3. 通知联动矩阵

### 3.1 通知规则

| 通知级别 | 触发场景 | 通知渠道 | 紧急度 |
|----------|----------|----------|--------|
| 🟢 信息 | 状态变更通知（SO 创建、工单下达等） | 站内信 | 低 |
| 🟡 重要 | 需要用户操作（待审批、待质检、待收货） | 站内信 + 邮件 | 中 |
| 🔴 紧急 | 异常事件（审批超时、质检不合格、库存不足） | 站内信 + 邮件 + 企微 | 高 |

### 3.2 完整通知联动表

| 序号 | 触发事件 | 来源模块 | 通知类型 | 通知渠道 | 通知对象 | 验证要点 |
|------|----------|----------|----------|----------|----------|----------|
| N1 | SO 创建 | Sales | 信息 | 站内信 + 企微 | 计划员、仓管 | 通知内容含 SO 单号和产品明细 |
| N2 | MRP 运算完成 | Planning | 信息 | 站内信 | 采购专员、生产主管 | 含需求数量和建议方案 |
| N3 | 采购 PO 发送 | Purchase | 信息 | 邮件 | 供应商 | 邮件含 PO 详情和交期 |
| N4 | 采购到货 | Purchase | 重要 | 站内信 + 邮件 | 质检员、仓管 | 含到货时间和物料清单 |
| N5 | 工单下达 | MES | 信息 | 站内信 + 企微 | 车间主任、仓管 | 含工单号和计划开工时间 |
| N6 | 生产领料需求 | MES | 重要 | 站内信 | 仓管 | 含物料清单和数量 |
| N7 | 生产完工报工 | MES | 信息 | 站内信 + 企微 | 质检员、仓管、计划员 | 含完工数量 |
| N8 | 质检完成（合格） | QMS | 信息 | 站内信 | 仓管、生产/采购 | 含检验结果和合格数量 |
| N9 | 质检不合格 | QMS | 🔴紧急 | 站内信 + 邮件 + 企微 | 质量主管、生产/采购经理 | 含不合格原因和处置建议 |
| N10 | 成品入库 | WMS | 信息 | 站内信 | 计划员、销售 | 含入库数量和仓位 |
| N11 | 发货出库 | WMS | 重要 | 站内信 + 邮件 + 企微 | 财务、销售、客户 | 含物流单号和预计到达 |
| N12 | 客户签收 | Sales | 重要 | 站内信 | 财务会计 | 触发开票 |
| N13 | 应收凭证生成 | FMS | 信息 | 站内信 | 销售经理 | 含应收金额和账期 |
| N14 | 开票完成 | FMS | 信息 | 站内信 + 邮件 | 销售、客户 | 含发票号和金额 |
| N15 | 收款到账 | FMS | 重要 | 站内信 | 销售、财务 | 含收款金额和核销状态 |
| N16 | 审批待办 | Workflow | 重要 | 站内信 + 邮件 | 审批人 | 含审批单号和摘要 |
| N17 | 审批超时催办 | Workflow | 🔴紧急 | 站内信 + 邮件 + 企微 | 审批人 + 其上级 | 含超时时长 |
| N18 | 库存不足预警 | WMS | 🔴紧急 | 站内信 + 企微 | 计划员、采购 | 含缺料明细和建议 |
| N19 | 信用额度超限 | FMS | 🔴紧急 | 站内信 + 企微 | 销售经理、财务 | 含客户和超额金额 |
| N20 | 成本差异预警 | FMS | 🟡重要 | 站内信 + 邮件 | 成本会计、生产经理 | 含差异金额和百分比 |

### 3.3 通知测试场景

| 场景编号 | 测试场景 | 验证方法 |
|----------|----------|----------|
| NT-E1 | 通知内容完整性 | 检查通知中是否包含所有必要字段（单号、金额、操作人） |
| NT-E2 | 通知不重复发送 | 同一事件不应产生重复通知（幂等性验证） |
| NT-E3 | 通知按级别路由 | 信息级只走站内信；紧急级走三渠道 |
| NT-E4 | 通知与事件时序 | 通知发出时间应在事件发生后 5s 内 |
| NT-E5 | 已读/未读状态 | 用户查看通知后标记为已读；未读通知在仪表盘显示计数 |
