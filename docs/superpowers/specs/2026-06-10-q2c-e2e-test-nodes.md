# Quote-to-Cash 全链路业务节点拆解

> **所属方案**：生产型 ERP 全链路 E2E 测试
> **本文档定义**：25+ 主线业务节点 + 异常分支场景

---

## Phase 1 — 销售域 (Sales)

### S1. 客户询价

| 属性 | 说明 |
|------|------|
| **输入** | 客户需求（产品型号、规格、数量、期望交期、目标价格） |
| **输出** | 询价记录（关联客户档案、产品主数据） |
| **触发下游** | → S2（销售报价） |
| **角色** | 销售专员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- S1-E1：客户信用冻结状态下提交询价 → 应提示信用异常
- S1-E2：询价产品不存在于产品主数据 → 应阻止或提示新建物料

---

### S2. 销售报价

| 属性 | 说明 |
|------|------|
| **输入** | 询价记录 + 价格表 + BOM 成本预估 + 税率 |
| **输出** | 报价单（含明细行：产品/数量/单价/折扣/税率/有效期/交付条款） |
| **触发下游** | → S3（报价审批）— 折扣率 > 15% 或金额 > 50万触发审批；否则直接生效 |
| **角色** | 销售专员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- S2-E1：报价折扣率超过阈值 → 自动触发审批流
- S2-E2：报价有效期过期后客户确认 → 应提示重新报价或续期
- S2-E3：报价产品 BOM 缺失 → 应阻止报价或标记待补充
- S2-E4：多次报价版本管理 → 版本号递增，旧版本自动失效

---

### S3. 报价审批

| 属性 | 说明 |
|------|------|
| **输入** | 报价单（触发条件满足时） |
| **输出** | 审批通过的报价单 / 驳回记录 |
| **审批类型** | 或签（销售经理 + 总经理，任一通过即生效） |
| **通过后动作** | 报价生效，可转销售合同 |
| **驳回后动作** | 退回销售专员修改 |
| **角色** | 销售经理 / 总经理 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- S3-E1：审批驳回后修改重提 → 验证重提流程是否正确
- S3-E2：审批超时自动升级 → 超过 24h 未审批自动升级到上级
- S3-E3：审批人委托代理 → 审批人请假期间委托他人审批
- S3-E4：提交人撤回 → 审批前撤回，审批流程终止

---

### S4. 销售合同

| 属性 | 说明 |
|------|------|
| **输入** | 客户确认的报价单 + 合同条款（付款条件/违约条款/交付条款） |
| **输出** | 销售合同（关联报价单，锁定价格和条款） |
| **触发下游** | → S5（销售订单） |
| **角色** | 销售主管 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- S4-E1：合同变更（价格调整/交期变更）→ 触发合同变更审批
- S4-E2：合同终止 → 验证是否正确清理关联的报价和预留

---

### S5. 销售订单

| 属性 | 说明 |
|------|------|
| **输入** | 合同 + 客户 PO（采购订单） |
| **输出** | 销售订单（SO）— 含明细行、交付计划、信用额度检查结果 |
| **触发下游** | → Phase 2（触发 MRP 运算）+ 库存预留 |
| **角色** | 销售专员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- S5-E1：客户信用额度不足 → 订单应被拦截或触发信用审批
- S5-E2：订单变更（数量/交期调整）→ 重新触发 MRP
- S5-E3：订单取消 → 验证库存预留释放、MRP 需求取消
- S5-E4：部分交付设置 → 允许分批发货的场景验证

---

## Phase 2 — 计划域 (Planning)

### P1. BOM 展开

| 属性 | 说明 |
|------|------|
| **输入** | SO 产品清单 |
| **输出** | 完整物料需求清单（多级 BOM 展开，含替代料） |
| **触发下游** | → P2（MRP 运算） |
| **角色** | 计划员 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- P1-E1：BOM 包含已停产物料 → 应提示替代料或阻止
- P1-E2：多级 BOM 循环引用 → 系统应检测并阻止

---

### P2. MRP 运算

| 属性 | 说明 |
|------|------|
| **输入** | BOM 展开 + 当前库存余额 + 在途采购 + 在制工单 + 安全库存参数 |
| **输出** | 净需求清单 + 采购建议（外购件）+ 生产建议（自制件） |
| **触发下游** | → Phase 3A（采购需求）+ Phase 3B（生产工单建议） |
| **角色** | 计划员 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- P2-E1：MRP 建议合并多个 SO 的同类物料需求 → 验证合并逻辑
- P2-E2：安全库存触发补货建议 → 验证补货数量计算
- P2-E3：MRP 重跑（需求变更后）→ 验证旧建议的取消和新建议的生成

---

### P3. 需求分解

| 属性 | 说明 |
|------|------|
| **输入** | MRP 运算结果 |
| **输出** | 采购需求清单（外购件）+ 生产工单建议（自制件） |
| **触发下游** | → PU1（采购申请）/ M1（工单下达） |
| **角色** | 计划员 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- P3-E1：需求分解后库存已变化 → 验证是否需要重新计算
- P3-E2：委外件识别 → 自制/外协/采购的判断逻辑验证

---

## Phase 3A — 采购域 (Purchase)

### PU1. 采购申请

| 属性 | 说明 |
|------|------|
| **输入** | 采购需求 + MRP 建议 |
| **输出** | 采购申请单（PR）— 含物料、数量、需求日期、建议供应商 |
| **触发下游** | → PU2（询价比价） |
| **角色** | 采购专员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- PU1-E1：MRP 建议被手动调整数量 → 验证调整记录
- PU1-E2：采购申请被驳回 → 退回修改流程

---

### PU2. 询价比价

| 属性 | 说明 |
|------|------|
| **输入** | PR + 供应商报价记录 |
| **输出** | 比价结果 + 推荐供应商 |
| **触发下游** | → PU3（采购审批） |
| **角色** | 采购专员 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- PU2-E1：只有一家供应商报价 → 验证是否触发单一来源审批
- PU2-E2：最低价供应商不在合格供应商名录 → 验证阻止逻辑

---

### PU3. 采购审批

| 属性 | 说明 |
|------|------|
| **输入** | 采购订单草案 + 比价结果 |
| **输出** | 审批通过的采购订单 / 驳回记录 |
| **审批类型** | 逐级（采购经理 → 财务总监），金额 > 10万或非常规物料触发 |
| **通过后动作** | 生成 PO 发送供应商 |
| **驳回后动作** | 退回采购修改 |
| **角色** | 采购经理 / 财务总监 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- PU3-E1：超额采购审批（超出 MRP 建议 20% 以上）→ 额外审批层级
- PU3-E2：审批驳回后修改供应商 → 验证是否需要重新比价

---

### PU4. 采购订单

| 属性 | 说明 |
|------|------|
| **输入** | 审批通过的采购申请 |
| **输出** | 采购订单（PO）— 含供应商、物料、数量、价格、交期、付款条件 |
| **触发下游** | → PU5（供应商发货通知） |
| **角色** | 采购专员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- PU4-E1：PO 变更（数量/价格/交期）→ 触发变更审批
- PU4-E2：PO 取消 → 验证到货通知清理、应付清理

---

### PU5. 到货通知

| 属性 | 说明 |
|------|------|
| **输入** | 供应商发货通知（ASN） |
| **输出** | 到货通知单 — 预计到货时间、物料清单、物流信息 |
| **触发下游** | → PU6（收货入库）+ 通知仓管准备收货 |
| **角色** | 采购专员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- PU5-E1：到货数量与 PO 不符 → 验证超交/短交的处理逻辑
- PU5-E2：分批到货 → 验证 PO 剩余数量更新

---

### PU6. 收货入库

| 属性 | 说明 |
|------|------|
| **输入** | ASN + 实物到货 |
| **输出** | 入库单 + 库存增加（待检状态）+ 触发来料质检 |
| **触发下游** | → QMS（来料检验）→ 质检通过 → 库存转可用 + FP1（应付确认） |
| **角色** | 仓管员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- PU6-E1：收货超交（超出 PO 数量）→ 验证是否允许及审批流程
- PU6-E2：收货短交 → 验证 PO 未交数量更新及催交通知
- PU6-E3：拒收（严重质量问题）→ 验证退货流程触发

---

## Phase 3B — 生产域 (MES)

### M1. 生产工单下达

| 属性 | 说明 |
|------|------|
| **输入** | 生产工单建议 + 产能检查 + 审批（紧急/超产能时） |
| **输出** | 已下达工单（WO）— 含 BOM 用量、工艺路线、计划开工/完工时间 |
| **触发下游** | → M2（生产领料） |
| **审批类型** | 单签（生产经理），紧急工单或超产能时触发 |
| **角色** | 生产主管 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- M1-E1：工单下达时物料库存不足 → 验证缺料预警和领料阻止
- M1-E2：工单暂停/恢复 → 验证状态流转和通知

---

### M2. 生产领料

| 属性 | 说明 |
|------|------|
| **输入** | WO + BOM 标准用量 |
| **输出** | 领料单 + 库存扣减（原材料仓） |
| **触发下游** | → M3（车间报工） |
| **角色** | 仓管员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- M2-E1：超领（超出 BOM 标准用量）→ 触发超领审批
- M2-E2：退料（多领退回）→ 验证库存回冲
- M2-E3：代料（使用替代物料）→ 验证替代料记录和成本差异

---

### M3. 车间报工

| 属性 | 说明 |
|------|------|
| **输入** | WO + 工艺路线 + 实际工时 + 实际用料 |
| **输出** | 工序完成记录 + 工时记录 + 用料记录 |
| **触发下游** | → M4（生产质检）— 最后一道工序报工后触发 |
| **角色** | 车间操作员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- M3-E1：工序报工数量与领料不匹配 → 验证用量差异预警
- M3-E2：工序跳过（免检工序）→ 验证自动流转到下一工序
- M3-E3：返工报工 → 验证返工工时和额外用料的记录

---

### M4. 生产质检

| 属性 | 说明 |
|------|------|
| **输入** | 报工完成 + 检验规范（AQL 抽样方案） |
| **输出** | 质检报告 — 合格/不合格/让步接收 |
| **触发下游** | 合格 → M5（成品入库）；不合格 → MRB 评审 |
| **角色** | 质检员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- M4-E1：质检不合格 → 触发 MRB（物料评审委员会）流程
- M4-E2：让步接收 → 验证客户通知和价格调整
- M4-E3：批量报废 → 触发报废审批（报废量 > 工单 5%）

---

### M5. 成品入库

| 属性 | 说明 |
|------|------|
| **输入** | 质检合格 + WO |
| **输出** | 成品入库单 + 库存增加（成品仓）+ 工单完工确认 |
| **触发下游** | → Phase 4（发货准备）+ F2（成本核算） |
| **角色** | 仓管员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- M5-E1：入库数量与报工数量不符 → 验证差异处理
- M5-E2：部分入库（分批完工）→ 验证工单部分完工状态

---

## Phase 4 — 仓储与发货 (WMS + Shipping)

### W1. 库存确认/锁定

| 属性 | 说明 |
|------|------|
| **输入** | SO + 成品库存余额 |
| **输出** | 可用库存确认 + 库存锁定（分配给 SO） |
| **触发下游** | → W2（拣货打包） |
| **角色** | 仓管员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- W1-E1：库存不足 → 触发缺货通知 + 回退等待生产完成
- W1-E2：库存已被其他 SO 锁定 → 验证分配优先级规则

---

### W2. 拣货打包

| 属性 | 说明 |
|------|------|
| **输入** | 发货通知 + 库存锁定记录 |
| **输出** | 拣货单 + 打包清单 |
| **触发下游** | → W3（发货出库） |
| **角色** | 仓管员 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- W2-E1：拣货数量错误 → 验证拣货校验和纠错流程
- W2-E2：打包时发现破损 → 触发质量复检

---

### W3. 发货出库

| 属性 | 说明 |
|------|------|
| **输入** | 打包完成 + 物流信息 |
| **输出** | 发货单 + 库存扣减 + 物流追踪号 |
| **触发下游** | → W4（客户签收）+ F1（应收确认） |
| **审批类型** | 单签（销售经理）— 超额发货或信用冻结客户触发 |
| **角色** | 物流专员 |
| **实现阶段** | 🟢 P0 |

**异常分支：**
- W3-E1：超额发货（超出 SO 数量）→ 触发审批
- W3-E2：发货后客户要求暂缓 → 验证拦截和库存处理

---

### W4. 客户签收

| 属性 | 说明 |
|------|------|
| **输入** | 物流送达确认（POD） |
| **输出** | 签收确认记录 → 触发收入确认 |
| **触发下游** | → F1（应收确认完成）+ F3（开票） |
| **角色** | 销售/物流 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- W4-E1：客户拒收 → 触发退货流程 + 库存回仓
- W4-E2：部分签收（短收）→ 验证差异处理和补发流程

---

## Phase 5 — 财务域 (FMS)

### 应收侧

#### F1. 应收确认

| 属性 | 说明 |
|------|------|
| **输入** | 发货单/签收确认 |
| **输出** | 应收账款凭证（AR）— 含金额、税额、客户、关联 SO |
| **触发下游** | → F2（成本核算） |
| **角色** | 财务会计 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- F1-E1：应收金额与 SO 金额不一致 → 验证差异预警
- F1-E2：客户信用额度超限 → 验证信用冻结是否阻止后续发货

---

#### F2. 成本核算

| 属性 | 说明 |
|------|------|
| **输入** | 领料成本（原材料）+ 人工成本（工时 × 费率）+ 制造费用（分摊规则） |
| **输出** | 产品单位成本 + 批次成本 + 毛利分析 |
| **触发下游** | → F3（开票，影响定价验证） |
| **角色** | 成本会计 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- F2-E1：实际成本超标准成本 > 10% → 触发成本差异预警
- F2-E2：在制品成本（WIP）计算 → 验证未完工工单的成本归集

---

#### F3. 开票

| 属性 | 说明 |
|------|------|
| **输入** | AR + 合同税率 |
| **输出** | 销售发票（含税额明细） |
| **触发下游** | → F4（收款） |
| **审批类型** | 逐级（财务主管 → 总经理）— 红字发票或金额差异 > 5% 触发 |
| **角色** | 财务会计 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- F3-E1：红字发票（退货/折让）→ 验证红冲逻辑和税务处理
- F3-E2：开票金额与应收不一致 → 验证差异处理

---

#### F4. 收款

| 属性 | 说明 |
|------|------|
| **输入** | 发票 + 客户付款记录 |
| **输出** | 收款凭证 + 现金/银行入账 |
| **触发下游** | → F5（对账核销） |
| **角色** | 出纳 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
- F4-E1：部分收款 → 验证部分核销逻辑
- F4-E2：多收/溢缴 → 验证预收账款处理
- F4-E3：外币收款 → 验证汇率转换和汇兑损益

---

#### F5. 对账核销

| 属性 | 说明 |
|------|------|
| **输入** | AR + 收款记录 |
| **输出** | 核销记录（应收冲收款）+ 信用额度释放 |
| **触发下游** | → F6（总账结算） |
| **角色** | 财务会计 |
| **实现阶段** | 🟡 P1 |

**异常分支：**
