# 质量评估与准出标准

> **所属方案**：生产型 ERP 全链路 E2E 测试
> **本文档定义**：Pass/Fail 标准、数据一致性校验、业财一体化验证、准出标准

---

## 1. 测试结果分级

### 1.1 断言结果分级

| 级别 | 标识 | 说明 | 影响 |
|------|------|------|------|
| **Critical** | 🔴 CRITICAL | 核心业务流阻断；数据不一致 | 必须修复，阻止发布 |
| **Major** | 🟠 MAJOR | 功能异常但不阻断主流程 | 应修复，建议阻止发布 |
| **Minor** | 🟡 MINOR | UI 问题、文案错误、体验不佳 | 可择机修复 |
| **Info** | 🔵 INFO | 建议优化项，非缺陷 | 记录，不影响发布 |

### 1.2 测试场景结果定义

| 结果 | 标识 | 条件 |
|------|------|------|
| **Passed** | ✅ | 所有断言通过，无异常 |
| **Failed** | ❌ | 任一 Critical/Major 断言失败 |
| **Blocked** | ⏸️ | 上游节点失败导致无法继续 |
| **Skipped** | ⏭️ | 未实现功能（P2 标记），跳过 |
| **Warning** | ⚠️ | Minor 断言失败，但主流程通过 |

## 2. Pass/Fail 标准

### 2.1 页面级验证标准

| 验证项 | Pass 条件 | Fail 条件 | 级别 |
|--------|-----------|-----------|------|
| **页面加载** | 页面在 5s 内完整渲染，无 JS 错误 | 白屏/JS 报错/加载超时 > 10s | Critical |
| **表单提交** | 提交成功，页面显示成功提示 | 提交后报错 500/数据未保存 | Critical |
| **数据展示** | 列表/详情页数据与数据库一致 | 数据缺失/排序错误/截断 | Major |
| **状态标签** | 状态标签与业务状态一致 | 状态标签显示错误 | Major |
| **操作权限** | 有权限用户可操作，无权限用户被阻止 | 无权限用户能执行操作 | Critical |
| **输入校验** | 非法输入被拦截并显示错误提示 | 非法输入被接受 | Major |
| **导航与返回** | 面包屑/返回按钮正常工作 | 返回后数据丢失或页面异常 | Minor |
| **响应时间** | CRUD 操作 < 3s | 操作响应 > 10s | Minor |

### 2.2 业务逻辑验证标准

| 验证项 | Pass 条件 | Fail 条件 | 级别 |
|--------|-----------|-----------|------|
| **BOM 展开** | 展开结果包含所有层级的物料 | 缺失层级或数量错误 | Critical |
| **MRP 运算** | 净需求 = 毛需求 - 在库 - 在途 - 在制 | 计算结果偏差 > 0.01 | Critical |
| **库存扣减** | 出库后库存余额正确减少 | 扣减数量错误或库存为负 | Critical |
| **库存预留** | SO 创建后库存被锁定 | 未预留或预留数量错误 | Critical |
| **成本核算** | 材料成本 + 人工 + 制造费用 = 总成本 | 成本分录不平衡 | Critical |
| **税额计算** | 税额 = 不含税金额 × 税率 | 税额计算错误（精度 < 0.01） | Critical |
| **审批流转** | 审批通过/驳回后状态正确变更 | 审批后状态未变更 | Critical |
| **事件触发** | 业务操作后对应事件在 5s 内触发 | 事件未触发或延迟 > 30s | Major |
| **通知发送** | 通知在事件触发后 5s 内送达 | 通知未送达或内容错误 | Major |

### 2.3 财务验证标准（业财一体化核心）

| 验证项 | Pass 条件 | Fail 条件 | 级别 |
|--------|-----------|-----------|------|
| **AR 凭证生成** | 发货后自动生成 AR 凭证，金额与发货单一致 | AR 未生成或金额不一致 | Critical |
| **AP 凭证生成** | 收货质检后自动生成 AP 凭证，金额与 PO 一致 | AP 未生成或金额不一致 | Critical |
| **成本分录平衡** | 每笔成本分录：借方合计 = 贷方合计 | 借贷不平衡 | Critical |
| **应收明细账** | AR 明细账 = 发票金额（含税）| 明细账与发票不一致 | Critical |
| **应付明细账** | AP 明细账 = 采购发票金额（含税）| 明细账与发票不一致 | Critical |
| **总账与明细账对平** | 总账余额 = 所有明细账余额之和 | 总账与明细账不平 | Critical |
| **核销准确性** | 核销金额 = 收款/付款金额；核销后 AR/AP 余额减少 | 核销后余额不正确 | Critical |
| **信用额度** | 收款核销后信用额度恢复 | 信用额度未恢复 | Major |
| **成本差异** | 实际成本与标准成本差异率 < 10% | 差异率 > 10% 且无预警 | Major |
| **期间完整性** | 所有凭证归属到正确的会计期间 | 凭证期间归属错误 | Critical |

## 3. 数据一致性校验

### 3.1 跨表一致性校验

以下是全链路测试完成后，必须验证的跨表数据一致性。

| 校验编号 | 校验项 | SQL 校验逻辑 | 预期结果 |
|----------|--------|-------------|----------|
| CHK-01 | **SO 与发货一致** | SO 明细数量 = 发货出库数量（部分交付场景除外） | 相等或 SO ≥ 发货 |
| CHK-02 | **SO 与 AR 一致** | AR 金额 = SO 明细金额 × (1 + 税率) | 相等 |
| CHK-03 | **PO 与收货一致** | 收货数量 ≤ PO 数量 | 收货 ≤ PO |
| CHK-04 | **PO 与 AP 一致** | AP 金额 = PO 明细金额 × (1 + 税率)（已收货部分） | 相等 |
| CHK-05 | **工单用料与 BOM 一致** | 实际领料数量 ≈ BOM 标准用量 × 报工数量（允许合理损耗） | 偏差 < 10% |
| CHK-06 | **工单成本归集完整** | 工单总成本 = 领料成本 + 人工成本 + 制造费用 | 相等 |
| CHK-07 | **库存余额正确** | 期初 + 入库 - 出库 = 期末库存 | 相等 |
| CHK-08 | **库存预留一致** | 库存预留总额 = 未发货 SO 的预留之和 | 相等 |
| CHK-09 | **总账借贷平衡** | 所有凭证的借方合计 = 贷方合计 | 相等 |
| CHK-10 | **AR 核销完整** | AR 已核销金额 + AR 未核销余额 = AR 总额 | 相等 |
| CHK-11 | **AP 核销完整** | AP 已核销金额 + AP 未核销余额 = AP 总额 | 相等 |
| CHK-12 | **审计日志完整** | 所有关键操作（创建/审批/状态变更）在审计日志中有记录 | 记录存在且完整 |

### 3.2 一致性校验执行时机

| 执行点 | 校验范围 |
|--------|----------|
| **每个节点完成后** | 该节点相关的 2-3 项校验（即时验证） |
| **每个 Phase 完成后** | 该 Phase 涉及的全部校验 |
| **全链路完成后** | 全部 12 项校验（最终验证） |

### 3.3 一致性校验脚本示例

```sql
-- CHK-07: 库存余额正确性校验
SELECT
  w.code AS warehouse,
  p.code AS product,
  i.opening_balance + COALESCE(r.received, 0) - COALESCE(s.shipped, 0) AS calculated,
  i.current_balance AS actual,
  i.current_balance - (i.opening_balance + COALESCE(r.received, 0) - COALESCE(s.shipped, 0)) AS diff
FROM inventory i
JOIN warehouse w ON i.warehouse_id = w.id
JOIN product p ON i.product_id = p.id
LEFT JOIN (SELECT warehouse_id, product_id, SUM(qty) AS received FROM stock_ledger WHERE type = 'in' GROUP BY 1,2) r ON ...
LEFT JOIN (SELECT warehouse_id, product_id, SUM(qty) AS shipped FROM stock_ledger WHERE type = 'out' GROUP BY 1,2) s ON ...
WHERE i.run_id = 'run001'
HAVING diff <> 0;
-- 预期结果：0 行（无差异）
```

## 4. 业财一体化验证

### 4.1 业务事件 → 会计分录映射

| 业务事件 | 应生成的会计分录 | 验证方法 |
|----------|-----------------|----------|
| **发货出库** | 借：应收账款 / 贷：主营业务收入 + 应交税金（销项） | AR 凭证金额 = 发货金额 × (1 + 税率) |
| **成本结转** | 借：主营业务成本 / 贷：库存商品 | 成本金额 = 产品单位成本 × 出库数量 |
| **收货入库** | 借：原材料 / 贷：应付账款（暂估） | 暂估金额 = PO 单价 × 收货数量 |
| **收到发票** | 借：应交税金（进项）/ 贷：应付账款（暂估）→ 应付账款 | 发票金额 = AP 金额 |
| **生产领料** | 借：生产成本（直接材料）/ 贷：原材料 | 领料金额 = 实际领料 × 移动加权单价 |
| **人工成本** | 借：生产成本（直接人工）/ 贷：应付职工薪酬 | 人工成本 = 实际工时 × 标准费率 |
| **制造费用** | 借：生产成本（制造费用）/ 贷：制造费用 | 分摊金额按预定分配率计算 |
| **成品入库** | 借：库存商品 / 贷：生产成本 | 入库金额 = 工单归集总成本 |
| **收到货款** | 借：银行存款 / 贷：应收账款 | 收款金额 = AR 核销金额 |
| **支付货款** | 借：应付账款 / 贷：银行存款 | 付款金额 = AP 核销金额 |

### 4.2 全链路财务闭环校验

全链路完成后，验证以下财务闭环：

```
收入侧:
  主营业务收入 = SO 金额（不含税）
  应收账款 = SO 金额（含税）
  已收款 = 客户付款
  应收余额 = 应收账款 - 已收款 = 0（全额收款场景）

成本侧:
  主营业务成本 = 产品单位成本 × 销售数量
  其中：直接材料 + 直接人工 + 制造费用 = 总成本
  毛利 = 主营业务收入 - 主营业务成本
  毛利率 = 毛利 / 主营业务收入

应付侧:
  应付账款 = 采购金额（含税）
  已付款 = 供应商付款
  应付余额 = 应付账款 - 已付款 = 0（全额付款场景）

总账平衡:
  所有凭证借方合计 = 所有凭证贷方合计
```

## 5. 准出标准

### 5.1 一票否决项（任一不通过则阻止发布）

| 编号 | 准出条件 | 验证方法 |
|------|----------|----------|
| GATE-01 | **Happy Path 全链路跑通**：报价 → 结算（含财务闭环）无阻断 | 全链路执行通过 |
| GATE-02 | **数据一致性**：12 项跨表校验全部通过 | 一致性校验脚本输出 0 差异 |
| GATE-03 | **总账借贷平衡**：所有会计分录借贷相等 | CHK-09 通过 |
| GATE-04 | **无 Critical 级别缺陷** | 缺陷列表中 Critical = 0 |
| GATE-05 | **库存不出现负数**：任何仓库、任何物料的库存余额 ≥ 0 | 库存快照验证 |

### 5.2 质量指标

| 指标 | 目标值 | 计算方式 |
|------|--------|----------|
| **主线场景通过率** | 100% | Passed / Total 主线场景 |
| **异常场景通过率** | ≥ 85% | Passed / Total 异常场景 |
| **Critical 缺陷数** | 0 | Critical 级别缺陷计数 |
| **Major 缺陷数** | ≤ 3 | Major 级别缺陷计数 |
| **数据一致性通过率** | 100% | CHK 校验通过数 / 总校验数 |
| **审批流正确率** | 100% | 审批场景通过数 / 总审批场景 |
| **通知送达率** | ≥ 95% | 通知实际送达数 / 应送达数 |
| **事件触发率** | 100% | 事件实际触发数 / 应触发数 |

### 5.3 测试报告内容

测试完成后生成的报告需包含以下内容：

| 报告章节 | 内容 |
|----------|------|
| **1. 执行摘要** | 总体通过率、关键指标、准出判定（Pass/Fail） |
| **2. 场景执行明细** | 每个场景的执行结果、耗时、截图 |
| **3. 接力链路追踪** | 从 SO 到结算的完整接力路径、每个节点的输出数据 |
| **4. 数据一致性报告** | 12 项校验的结果、差异明细 |
| **5. 财务闭环报告** | 收入/成本/应收/应付的完整对账表 |
| **6. 审批流报告** | 所有审批场景的流转记录、耗时统计 |
| **7. 事件与通知报告** | 事件触发/投递统计、通知送达统计 |
| **8. 缺陷清单** | 按级别排列的缺陷列表，含复现步骤和截图 |
| **9. 建议与优化** | 基于测试结果的改进建议 |

### 5.4 准出判定流程

```
测试执行完成
  │
  ├→ GATE-01 ~ GATE-05 是否全部通过？
  │     NO → ❌ 不准出，输出阻断原因
  │     YES ↓
  │
  ├→ 质量指标是否达标？
  │     NO → 🟡 有条件准出（附缺陷清单和修复计划）
  │     YES ↓
  │
  └→ ✅ 准出通过
       → 生成测试报告
       → 标记版本为"测试通过"
```
