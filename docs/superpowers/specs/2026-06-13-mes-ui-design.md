# MES 流程简化 — UI 设计规范

> 日期: 2026-06-13
> 状态: Draft v2（含 MES 使用者评审反馈）
> 依据: `docs/superpowers/specs/2026-06-13-mes-flow-simplification-design.md` v6
> 范围: 阶段 1-4 涉及的前端页面改造

## 0. 设计原则

1. **用户角色驱动** — 页面按角色（计划员、仓管员、车间工人）组织，不同角色看到不同的默认视图
2. **操作可逆** — 任何不可逆操作前必须有预览确认，关键操作提供回退路径
3. **进度可视化** — 关键实体（工单、批次）的生产进度一目了然，不需要逐行读表
4. **信息分层** — 摘要在前（卡片/进度条），明细在后（表格/Tab）
5. **现有组件复用** — `status-pill`、`info-card`、`form-section`、`data-table`、`status-tabs` 等

## 1. 需求池页面（mes_demand_pool.rs）

### 1a. 新增"待排程"视图 Tab

当前有两个视图：物料视图（按物料聚合）和明细视图。增加第三个视图：**待排程视图**。

计划员日常工作的核心场景是"看哪些订单要排产"，不是"看物料"。

```
┌──────────────────────────────────────────────────────────────────┐
│ [物料视图]  [明细视图]  [待排程视图]        搜索: [         🔍]  │
├──────────────────────────────────────────────────────────────────┤
│ 待排程: 15条  |  已排程: 8条  |  今日新增: 3条                   │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 🔴 SO20260610-005 电容器 XYZ-100  500件  交期 6/15 (3天后)      │
│    客户: XX电子  |  紧急  |  BOM ✓  Routing ✓                   │
│    [快速排程 →]                                                   │
│                                                                  │
│ 🟡 SO20260611-002 电阻器 R-200    300件  交期 6/20 (8天后)      │
│    客户: YY科技  |  普通  |  BOM ✓  Routing ✗                   │
│    [快速排程 →]                                                   │
│                                                                  │
│ ⚪ SO20260612-008 线圈 L-50       200件  交期 6/25 (13天后)     │
│    客户: ZZ集团  |  低    |  BOM ✗  Routing ✗                   │
│    [快速排程 →]  (缺少BOM，建议先补全主数据)                      │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

**设计要点**：
- 按交期紧急程度排序：🔴交期≤3天 > 🟡交期4-7天 > ⚪交期>7天
- 每条显示：来源订单号、产品名+编码、数量、交期天数、客户、紧急程度、BOM/Routing 状态
- "快速排程"直接跳到创建计划页面（自动选中该需求）
- 缺少主数据的需求行显示引导提示（"建议先补全主数据"）
- 顶部统计栏：待排程数、已排程数、今日新增数

**实现**：
- 新增 `get_demand_scheduling_view` handler 或在现有 `get_demand_pool_list` 中扩展 view_mode 参数
- 数据来源：`DemandSummary` + `boms`/`bom_routings` 关联查询
- 使用 `data-card` + 自定义卡片行渲染

### 1d. "创建并下达"需预览确认（阶段 3）

**关键原则：不可逆操作必须有预览。**

"创建并下达"不是直接执行，而是弹出预览 Modal：

```
点击"创建并下达" → 弹出 modal-overlay:

┌─ 确认下达预览 ──────────────────────────────────────────┐
│                                                          │
│ 将创建 3 个工单（已下达状态），包含工序、批次、领料单：  │
│                                                          │
│ ✅ 电容器  500件  6/14-6/24  装配车间A  工序: 5步       │
│ ✅ 电阻器  300件  6/15-6/25  (自动推断)  工序: 默认     │
│ ⚠️ 线圈   200件  6/16-6/26  物料不足(铜线 缺口50)      │
│                                                          │
│ 共计: 3 个工单  |  1000件  |  2 条预警                   │
│                                                          │
│              [返回修改]     [确认下达]                    │
└──────────────────────────────────────────────────────────┘
```

**设计要点**：
- 每行显示：产品、数量、排程日期、工作中心、工序来源（Routing名/默认）
- 有预警的行标黄色（物料不足、无 Routing 等），但不阻断
- 底部汇总：工单数、总数量、预警条数
- "返回修改"关闭 Modal，"确认下达"真正执行
- 执行后显示下达结果面板（见 §2c）

**实现**：
- "创建并下达"按钮改为 `hx-get` 加载预览 Modal（HTMX 局部刷新）
- Modal 内的"确认下达"才是 `hx-post` 真正提交

```
│ [→ 进入看板]                       │
└────────────────────────────────────┘
```

**车间看板（V2 方向）**：
- 入口：MES Dashboard → "车间看板"
- 按工作中心/工位筛选
- 显示：当前工位的待报工工序列表
- 操作：一键报工（输入完成量/报废量 → 提交）
- 不展示完整工单信息，只看工序级
- 面向车间工人，不需要权限管理复杂页面
