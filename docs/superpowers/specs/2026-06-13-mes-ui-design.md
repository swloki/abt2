# MES 流程简化 — UI 设计建议

> 日期: 2026-06-13
> 状态: Draft
> 依据: `docs/superpowers/specs/2026-06-13-mes-flow-simplification-design.md` v6
> 范围: 阶段 1-4 涉及的前端页面改造

## 0. 设计原则

1. **渐进式改造** — 不做页面重构，在现有页面结构上增量增加功能
2. **信息分层** — 关键信息（状态、警告、动作）突出，次要信息收纳
3. **操作引导** — 用户在正确的时机看到正确的操作按钮，不需要猜
4. **现有组件复用** — `status-pill`、`info-card`、`form-section`、`data-table` 等，不引入新 UI 组件

---

## 1. 需求池页面（mes_demand_pool.rs）

### 当前问题

- **无排程参数** — 创建计划时只有默认日期，缺少工作中心、优先级、Routing 信息
- **批量操作隐蔽** — batch_action_bar 存在但不明显，"创建生产计划"流程需跳转新页面
- **无 Routing/BOM 状态提示** — 计划员无法在需求池看到哪些产品有 Routing、有 BOM

### 改进建议

#### 1a. 物料卡片增加数据完整度指示器

在物料卡片行（`material_row`）上增加小型指示器，显示产品主数据完整度：

```
┌──────────────────────────────────────────────────┐
│ [产品图标] 电容器 XYZ-100   总需求: 500          │
│ 最早需求: 2026-06-15 (3天后) 🔴紧急              │
│                                                    │
│ ● BOM ✓   ● Routing ✗   ● 物料模式: 倒冲        │  ← 新增
│ [查看详情 ▼]              [创建计划 →]            │
└──────────────────────────────────────────────────┘
```

**实现**：
- `material_row()` 中增加一行状态指示
- BOM 状态：查 `bom_snapshots` 或 `boms` 表是否有已发布 BOM
- Routing 状态：查 `bom_routings` 表是否有关联
- 物料模式：从 `products.meta` 读 `material_consumption_mode`（阶段 2 才有，阶段 1 显示"默认"）

**数据来源**：`MaterialAggSummary` 模型需扩展 3 个字段：`has_bom: bool`, `has_routing: bool`, `material_mode: Option<String>`

#### 1b. 创建计划页面增加排程参数

当前 `mes_demand_pool_create.rs` 已有"默认排程参数"区域（排程开始/结束）。需要扩展：

```
┌─ 计划信息 ──────────────────────────────────────┐
│ 物料名称: [电容器 XYZ-100]  物料编码: [C-100]   │
│ 计划类型: [按单生产 ▾]  计划日期: [2026-06-13]   │
└──────────────────────────────────────────────────┘

┌─ 排程参数 ──────────────────────────────────────┐
│ 排程开始: [2026-06-14]  排程结束: [2026-06-24]   │
│ 工作中心: [装配车间 A ▾]     ← 新增             │
│ 优先级:   [2 ▾]              ← 已有，保留       │
│ 备注: [                                    ]     │
└──────────────────────────────────────────────────┘
```

**工作中心选择器**：
- 从 `work_centers` 表加载
- 带搜索的 `<select>`，使用现有 `form-select` 样式
- 可选（非必填），不填时系统自动根据 Routing 推断

#### 1c. 阶段 3：增加"确认并下达"一步操作

在需求池创建计划页面的操作栏，增加"创建并下达"按钮（阶段 3 实现）：

```
┌──────────────────────────────────────────────────┐
│ [取消]  [保存草稿]  [创建计划草稿]  [创建并下达 →]│ ← 新增
│                           (需 Routing + BOM 校验) │
└──────────────────────────────────────────────────┘
```

- "创建并下达"调用 `release_to_work_orders`，一步到位
- 按钮仅在所有选中需求的产品都有 BOM 时启用（否则 disabled + tooltip 提示）
- 提交后跳转到计划详情页，显示 `BatchReleaseResult` 结果

## 2. 生产计划详情页（mes_plan_detail.rs）

### 当前问题

- **操作分两步** — "确认计划"和"下达计划"是两个独立按钮，需手动操作两次
- **下达结果不可见** — 下达后直接重定向，看不到成功/失败的详细结果
- **计划行信息不足** — 缺少 Routing、工作中心、BOM 状态展示
- **无预警信息** — 缺失数据（无 Routing、无 BOM、物料不足）没有任何提示

### 改进建议

#### 2a. 计划行表格增加数据完整度列

```
┌─────┬──────────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│ 序号│ 产品     │ 数量 │ 开始 │ 结束 │ 优先 │ Routing │ 数据 │ 状态 │
│     │          │      │      │      │ 级   │ 状态    │ 完整 │      │
├─────┼──────────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ 1   │ 电容器   │ 500  │ 6/14 │ 6/24 │ 2    │ ✓ 有   │ ●●● │Planned│ ← 绿色
│ 2   │ 电阻器   │ 300  │ 6/15 │ 6/25 │ 3    │ ✗ 无   │ ●●○ │Planned│ ← 黄色
│ 3   │ 线圈     │ 200  │ 6/16 │ 6/26 │ 1    │ ✓ 有   │ ●●○ │Planned│ ← 黄色（无BOM）
└─────┴──────────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

**实现**：
- 表头增加"Routing"和"数据完整度"两列
- "数据完整度"用 3 个圆点（●●●）表示：BOM / Routing / 物料可用性
- 缺失项用空心圆（○），hover 时 tooltip 说明缺什么
- 行尾增加 `status-pill` 展示行状态（已有）

#### 2b. 下达按钮增强 — "确认并下达"合并操作

当前 `Confirmed` 状态时显示"下达计划"按钮。改为：

```
Draft 状态:
  [确认计划]          ← 保留，走两步流程

Confirmed 状态:
  [确认并下达 →]      ← 新增，一键到底
  (hover 显示: 将创建已下达的工单，包含工序、批次、领料单)

Released/InProgress 状态:
  [查看工单 →]        ← 跳转到关联工单列表
```

**"确认并下达"提交后**：
- 显示下达结果（成功/失败/警告），不用跳转
- 成功的行：状态变为 Released，显示关联工单号（链接）
- 失败的行：保持 Planned，显示错误原因 + [重试] 按钮

#### 2c. 下达结果展示（阶段 3）

```
┌─ 下达结果 ──────────────────────────────────────────────────┐
│ ✅ 成功: 3 个工单已创建并下达                                │
│ ❌ 失败: 1 个工单创建失败                                    │
│                                                             │
│ ✅ WO20260613001 电容器 500件  [→ 工单详情] [→ 批次详情]   │
│ ✅ WO20260613002 电阻器 300件  [→ 工单详情] [→ 批次详情]   │
│ ✅ WO20260613003 线圈   200件  [→ 工单详情] [→ 批次详情]   │
│ ❌ 线圈B  100件  失败原因: 未找到已发布BOM  [修正后重试]     │
│                                                             │
│ ⚠️ 预警: 电阻器无工艺路线（将使用默认工序"生产"）            │
│ ⚠️ 预警: 线圈物料不足: 铜线需要 200，可用 150，缺口 50      │
└─────────────────────────────────────────────────────────────┘
```

**实现**：
- `release_to_work_orders` 返回 `BatchReleaseResult`
- 页面渲染结果区域，使用 `info-card` + `status-pill` 组合
- 成功行用 `status-pill status-completed`（绿色）
- 失败行用 `status-pill status-rejected`（红色）
- 警告行用 `status-pill status-pending`（黄色）

## 3. 工单详情页（mes_order_detail.rs）

### 当前问题

- **信息量极少** — 只显示 8 个基础字段，缺少 BOM 快照、Routing、批次、领料单等关联信息
- **无反下达按钮** — 设计文档要求阶段 2 实现 `unrelease()`
- **缺少工序/批次/物料关联展示** — 工单下的核心子实体全部不可见

### 改进建议

#### 3a. 增加 BOM 快照 + Routing 信息展示

在工单信息卡片下方增加两个 info-card：

```
┌─ 工单信息 ────────────────────────────────────┐  ← 已有
│ 编号: WO20260613001  产品: 电容器 XYZ-100     │
│ 数量: 500  状态: Released  版本: 1            │
└────────────────────────────────────────────────┘

┌─ 生产数据 ────────────────────────────────────┐  ← 新增
│ BOM 快照: BS-20260613-001 (已冻结)             │
│ 工艺路线: RT-装配-001 (装配工艺路线)           │
│ 工序数: 5  工作中心: 装配车间 A               │
│ 物料模式: 倒冲                                │
│ 销售订单: SO20260610005 (客户: XX电子)         │  ← P6 修复后展示
└────────────────────────────────────────────────┘
```

**实现**：
- `get_order_detail` handler 中增加查询：BOM 快照信息、Routing 信息、关联销售订单
- 使用现有 `info-card` + `info-grid` 布局

#### 3b. 增加工序列表

```
┌─ 工序明细 ────────────────────────────────────────────────┐
│ 序号  工序名称     工作中心     计划量  完成量  状态       │
│ 1     来料检验     QC 工位      500     500     ✓ 已完成   │
│ 2     元件插入     装配线 A     500     300     ⏳ 进行中  │
│ 3     焊接         焊接线       500     0       ○ 待开始   │
│ 4     功能测试     测试台       500     0       ○ 待开始   │
│ 5     包装入库     包装区       500     0       ○ 待开始   │
└───────────────────────────────────────────────────────────┘
```

- 数据来源：`WorkOrderRouting`（通过 `work_order_id` 查询）
- 状态用 `status-pill`：Pending(灰)、InProgress(蓝)、Completed(绿)

#### 3c. 增加生产批次/领料单/倒冲记录关联 Tab

```
┌─ 关联单据 ────────────────────────────────────────┐
│ [生产批次] [领料单] [倒冲记录] [报工记录]          │ ← Tab 切换
├────────────────────────────────────────────────────┤
│ 批次号       数量   状态        工序进度           │
│ B20260613001  500   进行中      2/5 (焊接中)      │
│                                                   │
│                    [→ 查看批次详情]                │
└───────────────────────────────────────────────────┘
```

**实现**：
- 使用现有 `status-tabs` 组件
- 每个 Tab 内容通过 HTMX 按需加载（`hx-get` 到各自的 TypedPath + `?work_order_id=X`）
- 链接到各自的详情页

#### 3d. 增加"反下达"按钮（阶段 2）

```
Released 状态的工单：
  [关闭工单]  [反下达 ↩]  [取消工单]

反下达按钮:
  - 样式: btn-default（非主要操作，不抢视觉）
  - 确认框: "反下达将回退工单到草稿状态，同时取消领料单、释放库存预留、删除批次。确认反下达？"
  - 前置校验失败时: disabled + tooltip 说明原因（已开工/已领料）
```

#### 3e. 无 Routing 工单的默认工序提示

当工单使用了虚拟默认工序时，在工序列表上方显示提示：

```
┌─────────────────────────────────────────────────────┐
│ ⚠️ 该产品无已定义的工艺路线，使用默认工序"生产"。    │
│ 建议在产品主数据中维护工艺路线。                      │
│ [→ 维护工艺路线]                                     │
└─────────────────────────────────────────────────────┘
```

- 样式：`background: #fffbeb; border-left: 3px solid #f59e0b; padding: 12px 16px`
- 链接到该产品的 Routing 创建页面

## 4. 产品详情页（product_detail.rs）

### 改进建议

#### 4a. 增加"生产配置"信息区（阶段 2）

在产品详情页增加一个 `info-card`：

```
┌─ 生产配置 ─────────────────────────────────────┐
│ BOM:          [BOM-001 v2] (已发布) [→ 查看]   │
│ 工艺路线:     [RT-装配-001] (已发布) [→ 查看]  │
│ 物料消耗模式: 倒冲  [编辑]                      │  ← 阶段 2
│ 超额容差:     5%        [编辑]                  │  ← 阶段 3
│                                               │
│ ⚠️ 无工艺路线: 该产品尚未维护工艺路线            │  ← 无 Routing 时
│ [→ 创建工艺路线]                                │
└────────────────────────────────────────────────┘
```

**实现**：
- 查询 `boms`（已发布 BOM）+ `bom_routings`（关联 Routing）+ `products.meta`
- 使用 `info-card` + `info-grid` 布局
- 无数据时显示警告提示 + 引导创建

## 7. 实施映射

| 设计文档阶段 | UI 改动 | 涉及页面 |
|-------------|---------|---------|
| **阶段 1（止血）** | 无 UI 改动（纯后端修复） | — |
| **阶段 2（安全网+picking）** | 反下达按钮、工单详情增加工序/批次/领料关联、产品详情增加生产配置区、物料模式切换 | mes_order_detail, product_detail |
| **阶段 3（一键贯通）** | 计划详情"确认并下达"按钮+下达结果展示、物料预检 warnings、超额容差提示 | mes_plan_detail, mes_demand_pool_create |
| **阶段 4（前端+排程）** | 需求池排程参数+工作中心选择、数据完整度指示器、Dashboard 数据质量入口、工单列表来源标记 | mes_demand_pool, mes_demand_pool_create, mes_dashboard, mes_order_list |
