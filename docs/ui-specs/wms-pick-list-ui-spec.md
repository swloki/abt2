# 销售拣货单（PickList）UI 需求设计

> 关联：#93 Phase 3 / `wms/pick_list` 后端。本文档为 Open Design 原型的设计输入。
> 现有参照：ABT 发货单页（`/admin/wms/shipping`）、出库管理风格（UnoCSS + HTMX + Maud SSR）。

## 1. 页面定位

**仓库岗的拣货工作台**。销售发货单（outbound）审核确认后生成拣货单，仓库人员在此核对产品、到库位拣货、录入实拣数量，完成后驱动发货出库。

归属：WMS 出库管理下。角色：仓库拣货员。

## 2. 核心场景

1. 仓库收到一张已确认的发货单，需要为其中的产品拣货
2. 逐行核对：产品编码/名称、请求数量、当前库存可用量、建议库位
3. 到库位实物拣货，录入**实拣数量**（默认 = 请求数量满拣，支持部分拣/欠货）
4.（可选）指定实际拣货库位
5. 全部拣完 → 提交「完成拣货」→ 拣货单 Draft → Picked → 可继续发货（ship）

## 3. 页面结构

### 3.1 顶部信息卡
- 拣货单号（PK-YYYY-MM-SEQ）+ 状态徽章（拣货中 Draft / 已完成 Picked）
- 关联发货单号（SR-，可点击跳转）+ 客户名称
- 拣货人 + 拣货时间（Picked 后显示）
- 主操作按钮：**「完成拣货」**（Draft 时可用，Picked 后隐藏/置灰）

### 3.2 拣货明细表（核心）
| 列 | 说明 |
|---|---|
| 行号 | # |
| 产品 | 编码 + 名称（+ 规格） |
| 请求数量 | 来自发货单（只读） |
| 库存可用量 | 该产品在仓库的 ATP（只读，辅助判断够不够拣） |
| 建议库位 | 系统按 FIFO/FEFO 建议的库位（只读提示） |
| **实拣数量** | **数字输入**（默认=请求数量，可改为部分拣） |
| 拣货库位 | 下拉选择（该产品在该仓库的库位列表，可选） |

### 3.3 底部操作区
- 「完成拣货」主按钮（提交全部行）
- 取消/返回

## 4. 交互细节

- **实拣数量输入**：默认填入请求数量（满拣）。仓库员可下调（部分拣，余量作欠货）。校验：≥0，≤请求数量（不允许超拣，超拣走单独流程）。
- **拣货库位**：下拉，选项 = 该产品在该仓库有库存的库位。默认选中建议库位。
- **行级即时反馈**：实拣 < 请求数量时，行高亮提示「欠货 N」（琥珀色）。
- **提交校验**：所有行实拣数量已填；至少一行 > 0。
- **HTMX 提交**：`hx-post` 完成拣货端点，成功后状态切 Picked、表格转只读、顶部状态徽章更新。
- **已完成（Picked）态**：全部只读，显示实拣数量/库位，无编辑。

## 5. 状态流转

| 状态 | 含义 | 可操作 |
|---|---|---|
| 拣货中（Draft） | 待录入/已部分录入 | 编辑实拣数量、完成拣货、取消 |
| 已完成（Picked） | 拣货完成，待发货 | 只读；发货单可继续 ship |

## 6. 参考范式

- **ERPNext Pick List**：拣货单独立单据，记录 picked_qty + 库位
- **Odoo stock.picking**：assigned→done，扫码拣货、数量校验、欠货 backorder

## 7. ABT 视觉风格约束（设计须遵守）

- UnoCSS 原子类（无内联 style），语义色 token（success/warning/danger/bg/fg/border）
- 状态徽章用 `status-pill` 模式
- 表格用 `data-table` shortcut 风格
- 卡片用 `data-card` shortcut
- 数字输入右对齐，数量用 `fmt_qty` 格式化
- 中文界面，信息密度中等，仓库岗实操友好（大字号数量输入、清晰行高）
