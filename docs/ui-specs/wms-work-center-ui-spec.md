# WMS 作业中心 UI 需求设计

> 关联：`docs/uml-design/wms-work-center.md`（WorkCenterService）。本文档为 Open Design 原型设计输入。
> 参照：Odoo Inventory Operations 看板、ERPNext 仓库待办。ABT 风格（UnoCSS + HTMX + Maud SSR）。

## 1. 页面定位

**仓库岗的作业首页 / 待办看板**。仓库人员一进系统，在此看到所有待处理的仓库作业（按业务环节分区），知道先做什么、有多少、点进去处理。

## 2. 核心场景

1. 仓库员登录 → 直达作业中心
2. 一眼看到各环节待办数量（待收货 5、待拣货 3、待发货 2…）
3. 数量 > 0 的分区高亮（红/橙徽章吸引注意）
4. 点某分区 → 跳转该域的待处理列表（带状态筛选）
5. 处理完回到作业中心，数量实时更新（HTMX 局部刷新）

## 3. 页面结构

### 3.1 顶部
- 标题「仓库作业中心」+ 日期 + 当前班次/操作员（可选）
- 一句话提示「您有 N 项待办」（N = 总待办数）

### 3.2 待办分区卡片网格（核心，按业务流顺序排列）

每张卡片：
- 图标（按环节：收货↓/质检🔍/拣货📦/发货🚚/领料📋/调拨🔄/盘点✅）
- 环节名称（待收货 / 待质检 / 待拣货 / 待发货 / 待领料 / 待调拨 / 待盘点）
- **数量徽章**（大号数字；>0 用 warning/danger 色，=0 用 muted）
- 简短说明（如"PO 到货待接收"）
- 整卡可点（跳转该域列表）

卡片按业务流向排列（收货 → 质检 → ... → 发货），引导作业顺序。

### 3.3 可选：最近动态 / 紧急项
- 底部一行「紧急 / 临期」提示（如逾期收货、待发货超期等，后续增强）

## 4. 示例数据

| 分区 | 数量 | 说明 |
|---|---|---|
| 待收货 | 5 | PO 到货待接收 |
| 待质检 | 2 | 来料待检 |
| 待拣货 | 3 | 发货单待拣货 |
| 待发货 | 2 | 已拣货待发出 |
| 待领料 | 0 | —（muted） |
| 待调拨 | 1 | 调拨在途 |
| 待盘点 | 0 | —（muted） |

## 5. 交互（HTMX）

- 卡片点击 → 跳转该域列表（带 ?status=对应pending 筛选）
- 数量徽章 >0 高亮（warning/danger），=0 灰色（muted）
- 作业中心本身可定时或事件刷新（HTMX hx-trigger="refresh" from:body，其他页操作后回到作业中心数量更新）

## 6. 视觉要求（ABT 风格）

- UnoCSS 原子类（无内联 style），语义色 token
- 卡片网格（grid，响应式 2-4 列），每卡 `data-card` 风格（白底 + border + 圆角 + 轻阴影）
- 数量徽章大号、醒目（tabular-nums），是页面视觉焦点
- 卡片 hover 高亮（border-accent / 轻微上浮）
- 业务流向排列（左→右 / 上→下 = 收货→发货），引导作业顺序
- 中文界面，仓库岗实操友好（大字号、高对比、清晰图标）
- =0 的卡片降饱和（muted）但不隐藏（让用户知道这类当前无待办）
