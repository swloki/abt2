ABT ERP — Module Interface Dependencies

设计原则

① 接口先行 — 先定义 Service trait 契约，再实现
② 模块解耦 — 通过 DocumentType 枚举，共享层不引用具体业务类型
③ 事件驱动 — 业务模块通过 DomainEventBus 发布事件，handler 自动响应
④ 状态机统一 — StateMachineService 管理所有单据生命周期，替换分散的 if-else
⑤ 质量关卡硬门 — 检验不合格自动阻断下游流转（数据层面，非流程软约束）
⑥ 业财一体 — 从第一天起记录成本，统一 cost_entries 分类账
⑦ 流程编排 — Workflow Engine 处理审批、多步骤编排和 Saga 补偿
⑧ 全链路审计 — 业务 Service 同事务内直接调用 AuditLogService，字段级 diff + 请求上下文，零遗漏可追溯

事件驱动集成模式

┌──────────────────────────────────────────────────────────────────────┐
│ SalesOrderService.confirm() // 业务模块只关心核心逻辑 │
│  ├── StateMachine.transition("Confirmed") // 推进状态 │
│  ├── InventoryReservation.reserve(SOFT) ← 同步强一致 │
│  └── EventBus.publish(SalesOrderConfirmed) // 写 Outbox │
│        ├── CostEntryHandler → record cost ← 异步 │
│        ├── DocLinkHandler → link QUO→SO ← 异步 │
│        └── WorkflowTriggerHandler → start WF ← 异步 │
└──────────────────────────────────────────────────────────────────────┘
同步强一致：库存预留、质量关卡硬门 — 调用者事务内直接执行，失败即回滚
异步 Outbox：成本记录、文档链接、Workflow 触发、通知 — 业务事务内写 Outbox + NOTIFY，EventProcessor 后台消费

跨模块接口调用链

EventBus 事件分发 V2

■ 同步强一致（调用者事务内直接调用）
SalesOrderConfirmed → InvRes.reserve(SOFT)
SalesOrderShipped → InvRes.fulfill(释放SOFT) + InvTransaction.record
WorkOrderReleased → InvRes.reserve(HARD) + MaterialRequisition.create
ProductionReceiptConfirmed → InvRes.fulfill(释放HARD) + Backflush.execute

■ 异步 Outbox（写 Outbox + EventProcessor 后台消费）
SalesOrderConfirmed → CostEntryHandler + DocLinkHandler + WorkflowTriggerHandler
SalesOrderShipped → CostEntryHandler(COGS)
ArrivalNoticeReceived → CostEntryHandler(材料成本) + InvResReleaseHandler(释放安全库存)
ProductionReceiptConfirmed → CostEntryHandler(成品) + DocLinkHandler
OutsourcingReceived → CostEntryHandler(外协成本) + DocLinkHandler
InspectionFailed → MRBHandler(严重时创建MRB)

■ 审计（独立路径，非事件驱动）
业务 Service 在同一事务内直接调用 AuditLogService.record()，携带字段级 diff 和请求上下文，与 EventBus 平行独立

Sales → WMS (发货出库)

SalesOrderService.confirm()
  → StateMachine.transition("Confirmed")
  → EventBus.publish(SalesOrderConfirmed)
    → InventoryReservationService.reserve(Soft, TTL=7天)

ShippingRequestService.confirm()
  → InspectionResultService.is_passed(OQC) // 质量关卡硬门
  → InventoryReservationService.fulfill(释放SOFT预留)
  → CostEntryService.create(销售成本: 借COGS/贷存货)
  → InventoryTransactionService.record(SalesShipment)

Purchase → WMS (来料入库)

PurchaseOrderService.confirm()
  → ArrivalNoticeService.create() 触发来料通知

ArrivalNoticeService.confirm() 入库确认
  → InspectionResultService.is_passed(IQC) // 质量关卡硬门
  → EventBus.publish(ArrivalNoticeReceived)
    → CostEntryService.create(材料成本: 借存货/贷应付)
    → InventoryReservationService.fulfill(释放安全库存预留)

MES → WMS (生产全流程)

WorkOrderService.release() 工单下达
  → StateMachine.transition("Released")
  → EventBus.publish(WorkOrderReleased)
    → InventoryReservationService.reserve(Hard, 锁定物料)
    → MaterialRequisitionService.create_for_work_order() 触发领料

ProductionReceiptService.confirm() 完工入库
  → EventBus.publish(ProductionReceiptConfirmed)
    → InventoryTransactionService.record(ProductionReceipt)
    → CostEntryService.create(成品入库成本)
    → BackflushService.execute() 触发倒冲
    → InventoryReservationService.cancel(释放剩余Hard)

WorkOrderService.outsource() 委外加工
  → OM.OutsourcingOrderService.create_from_work_order() // 跨模块调用
    → OM 接管后续流程：发外协、追踪、收货、质检、结算

OM 委外 → WMS/MES/QMS

OutsourcingOrderService.send() 发外协
  → EventBus.publish(OutsourcingSent)
    → InventoryTransferService.transfer(物料→虚拟仓)

OutsourcingOrderService.receive() 外协收货
  → EventBus.publish(OutsourcingReceived)
    → InspectionResultService.create(IQC) // 触发来料检验
    → CostEntryService.create(外协服务成本)

OutsourcingOrderService.convert_to_internal() 转自制
  → EventBus.publish(OutsourcingConvertedToInternal)
    → MES.WorkOrderService.create() 创建内部工单

QMS 质量关卡 = 状态门

IQC 硬门 → WMS.ArrivalNoticeService.confirm() 前检查 is_passed(ArrivalNotice)
IPQC 自动触发 → MES.WorkOrderRouting.is_inspection_point == true 时自动创建检验
OQC 硬门 → Sales.ShippingRequestService.ship() 前检查 is_passed(ShippingRequest)
MRB 不良处置 → EventBus(MRBDispositioned) → CostEntry(返工/报废成本) + WMS库存调整
RMA 客诉追溯 → EventBus(RMACreated) → DocumentLink 全链路追溯

FMS 财务 × 业财一体

CashJournalService → EventBus(CashJournalConfirmed) → 所有现金流入流出
WriteOffService → EventBus(WriteOffCompleted) → 销售回款按出库单核销 | 采购付款按入库单核销
ExpenseReimbursementService → 审批通过 → 自动生成 CashJournal(Outflow)
CostAccountingService → 聚合查询 CostEntry：料/工/费完整归集 + 利润中心 P&L
