业务流转

Quotation 报价 ──DERIVED_FROM──→ SalesOrder 销售订单 ──TRIGGERS──→ ShippingRequest 发货申请 ──RECONCILES──→ Reconciliation 月对账单
↗ SalesReturn 销售退货 （从 ShippingRequest 逆向触发）

Quotation 报价单

功能：结合 BOM 成本规范商务询报价，锁定订单初始利润

entity Quotation {
  id: i64 PK
  doc_number: String // QUO-2026-05-xxxxx
  customer_id: i64 FK→customers (Master Data)
  contact_id: i64 FK→contacts
  sales_rep_id: i64 // 业绩归属人，创建时锁定，不随 Customer.owner_id 漂移
  quotation_date: Date
  valid_until: Date // 报价有效期
  status: QuotationStatus
  total_amount: Decimal(20,4)
  total_cost: Decimal(20,4)
  estimated_margin: Decimal(5,2) // 预估毛利率%
  payment_terms: String // 付款条件
  delivery_terms: String // 交货条款
  remark: String
  operator_id: i64 FK→users
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

entity QuotationItem {
  id: i64 PK
  quotation_id: i64 FK→quotations
  line_no: i32
  product_id: i64 FK→products
  description: String
  quantity: Decimal(18,6)
  unit: String
  unit_price: Decimal(18,6)
  unit_cost: Decimal(18,6) // BOM展开成本
  discount_rate: Decimal(5,2)
  amount: Decimal(20,4)
  delivery_date: Date?
}

enum QuotationStatus {
  DRAFT, SENT, ACCEPTED, REJECTED, EXPIRED
  }

共享层交互：DocumentSequence 生成编号 | CostEntry 无直接写入（报价阶段不产生实际成本）

ReconciliationStatus

enum ReconciliationStatus {
  DRAFT // 草稿 — 自动聚合发货明细，可手动调整
  SENT // 已发送 — 发送给客户确认

  }

错误模型 & 业务校验规则

DomainError — 统一错误模型

所有 Service 方法返回 Result<T, DomainError>，禁止 anyhow::Result。ServiceError 标记 deprecated，新代码统一使用 DomainError。

enum DomainError { NotFound(String), // 实体不存在 Duplicate(String), // 唯一约束冲突 PermissionDenied(String), // 权限不足 BusinessRule(String), // 业务规则违反（状态转换、数量校验等） Validation(String), // 输入校验失败（字段格式、必填缺失） ConcurrentConflict, // 乐观锁版本冲突 Internal(#[from] anyhow::Error), // 兼容老代码 } // gRPC 映射规则（handler 层）： // NotFound → tonic::Status::NotFound // Duplicate → tonic::Status::AlreadyExists // PermissionDenied → tonic::Status::PermissionDenied // BusinessRule → tonic::Status::FailedPrecondition // Validation → tonic::Status::InvalidArgument // ConcurrentConflict → tonic::Status::Aborted // Internal → tonic::Status::Internal

状态流转规则 — 合法转换矩阵

QuotationStatus

Draft → Sent | submit()
Draft → Expired | expire()
Sent → Accepted | accept()
Sent → Rejected | reject()

SalesOrderStatus (头状态 — 交付进度)

Draft → Confirmed | confirm() + 预留 + 履行计划 + 需求事件
Confirmed → ReadyToShip | recalc_header_status() 全行 Allocated（库存已补足、未发货）
ReadyToShip → PartiallyShipped / Shipped | SR.ship() + recalc_header_status()
Shipped → Completed | complete() open_qty = 0 防御 cancelled_qty
Draft / Confirmed / ReadyToShip → Cancelled | cancel()

ShippingStatus

Draft → Confirmed | confirm()
Confirmed → Picking | pick()
Picking → Shipped | ship()
Draft / Confirmed → Cancelled | cancel()

ReturnStatus

Draft → Confirmed | approve()
Confirmed → Received | receive()
Received → Inspecting | inspect() 触发 QMS 质检
Inspecting → Completed | complete() 按 disposition 处理
Inspecting → Cancelled | cancel() 质检不合格
Draft → Rejected | reject()

ReconciliationStatus

Draft → Sent | send() 发送客户 → Confirmed | confirm() 客户确认 → Settled | settle() 结算核销
Sent / Confirmed → Disputed | dispute() 提出异议
Disputed → Draft | reopen() 打回重做
Disputed → Settled | force_settle() 差异结算

SalesOrderLineStatus (行状态)

Pending → Allocated | 库存已硬预留
Pending → Producing | AcquireChannel=SelfProduced, 生产订单已创建
Pending → Purchasing | AcquireChannel=Purchased, 采购单已创建
Allocated / Producing / Purchasing → Shipped | SR.ship() 发货确认
Pending → Cancelled | cancel_line()

FulfillmentLineStatus

Pending → Allocated | 库存硬预留成功
Pending → Producing | 生产订单创建
Pending → Purchasing | 采购单创建
Allocated / Producing / Purchasing → Fulfilled | 下游单据完成

业务校验规则 — 操作前置条件

Quotation 校验

  • create/update: valid_until 必须 > 当前日期
  • update: 仅 Draft 状态可编辑
  • submit: 必须包含至少一个 Item
  • accept/reject: 仅 Sent 状态可操作
  • expire: 系统定时扫描 valid_until < now 的 Sent 报价单

SalesOrder 校验

  • create_from_quotation: 报价单状态必须为 Accepted
  • create_from_quotation: 报价单 valid_until 不得已过期
  • update_header: 仅 Draft 状态可修改
  • confirm: 触发 InventoryReservation.reserve(Soft) + create_fulfillment_plan + DemandCreated事件
  • complete: open_qty = quantity - shipped_qty - cancelled_qty = 0 防御 cancelled_qty
  • cancel_line: 仅 Pending 状态行可取消，自动 recalc_header_status
  • recalc_header_status: 幂等重算，根据行状态聚合推导头状态

ShippingRequest 校验

  • create_from_order: 订单状态 ≥ Confirmed
  • create_from_order: requested_qty ≤ order_qty - shipped_qty
  • confirm: QMS OQC hard gate 必须通过
  • ship: 自动更新 order_item.shipped_qty
  • ship: 触发 InvRes.fulfill + CostEntry(COGS)

SalesReturn 校验

  • create: 对应发货单状态必须为 Shipped
  • create: returned_qty ≤ shipped_qty - already_returned_qty
  • approve: 仅 Draft 状态
  • complete: 更新 order_item.returned_qty
  • complete: 触发 CostEntry(reverse) + QMS.RMAService

Reconciliation 校验

  • create: 同一 customer_id + period 不允许重复（唯一约束）
  • create: 自动聚合该客户+期间内所有 Shipped 的发货明细
  • confirm: 所有 Item 必须已确认（confirmed = true）或已调整
  • settle: 触发 FMS CashJournal + WriteOff 核销
