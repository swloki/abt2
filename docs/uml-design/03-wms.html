WMS Module — UML Class Diagram

核心业务流

ArrivalNotice 来料通知 → IQC检验 → Putaway 上架入库 → StockLedger 库存账
← Pick/Ship 拣货出库 ← MaterialRequisition 生产领料 ← Backflush 系统倒冲
Transfer 调拨 | FormConversion 形态转换 | CycleCount 盘点 | InventoryLock 锁库

InventoryService 库存操作服务（实现）

高层库存操作门面，包装 InventoryTransactionRepo + StockLedgerRepo，映射旧 abt 的 proto 接口

struct StockOperationResult {
  transaction_id: i64 // inventory_transactions.id
  stock_ledger_id: i64 // stock_ledger.id
  product_id: i64
  warehouse_id: i64
  zone_id: i64
  bin_id: i64
  before_qty: Decimal
  after_qty: Decimal
  change_qty: Decimal
}

写操作流程（stock_in / stock_out / adjust / set_quantity）：
  1. StockLedgerRepo::find_by_location() → before_qty
  2. InventoryTransactionRepo::insert() → transaction 记录
  3. StockLedgerRepo::upsert() → 更新库存账
  4. StockLedgerRepo::find_by_location() → after_qty + stock_ledger_id
  5. 返回 StockOperationResult

transfer：执行两次写操作（out 负数量 + in 正数量）
set_quantity：先读取当前 quantity，计算 delta = target - current
读操作：stock_ledger JOIN products/bins/zones/warehouses 动态过滤 + 分页
日志查询：inventory_transactions JOIN products/bins/zones/warehouses
// proto location_id 映射到 abt-core bin_id（通过 InventoryRepo.resolve_bin 解析）

InventoryCascadeService 级联库存查询

BOM 展开查询：给定一个成品，展开其所有 BOM 子件并汇总各子件的当前库存总量。用于快速判断成品是否可排产、缺料分析

struct CascadeInventoryResult {
  product_id: i64
  product_code: String
  product_name: String
  bom_groups: Vec<BomCascadeGroup> // 按 BOM 分组
}

struct BomCascadeGroup {
  bom_id: i64
  bom_name: String
  children: Vec<ChildNodeInventory>
}

struct ChildNodeInventory {
  node_id: i64 FK→bom_nodes
  product_id: i64
  product_code: String
  product_name: String
  unit: String?
  quantity: Decimal(18,6) // BOM 用量
  total_stock: Decimal // 当前库存汇总
  loss_rate: Decimal // 损耗率
  order: i32 // BOM 节点排序
  parent_node_id: i64?
}

trait InventoryCascadeService {
  // ctx: &ServiceContext, db: PgExecutor
  cascade_inventory(ctx, db,
    product_id: i64?, product_code: String?,
    max_results: i32)
    → CascadeInventoryResult
    // product_id/product_code 二选一
    // 查找所有 BOM → 展开子节点 → 聚合库存
    // max_results 限制返回节点数（防大 BOM 爆炸）
}

查询逻辑：Product → bom_nodes JOIN boms（产品作为子件被引用）→ 按 bom_id 分组 → DFS 展开子节点 → stock_ledger 聚合库存 | max_results 限制返回 BOM 数量
实现细节：InventoryCascadeServiceImpl 直接 SQL 查询 bom_nodes + boms，不经过 BomQueryService。分 7 步：查产品 → 查 BOM 引用 → 加载节点 → 查产品名称 → 构建树 → 聚合库存 → 填充结果
事务模式：None（只读查询，使用 ServiceContext + PgExecutor） | 用途：排产前缺料分析、采购建议、BOM 库存看板

DocumentLink → 来料→采购单 | 领料→工单 | 出库→发货申请 | 倒冲→工单

CostEntry → 入库记材料成本 | 出库记销售成本 | 倒冲差异记损耗成本 | 形态转换成本分摊
StateMachine → AN/MR/BF/CC/TRF/FC/LCK 所有单据状态推进
QMS → 来料入库前 IQC 硬门 is_passed(ArrivalNotice) | 不合格阻断入库
