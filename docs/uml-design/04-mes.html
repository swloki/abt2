双模式排产 + 工单管理 + 工序报工 + 委外统一 + 用量差异闭环

MES Module — UML Class Diagram

核心业务流

ProductionPlan 生产计划 → WorkOrder 工单 → WorkOrderRouting 工序 → WorkReport 报工 → ProductionReceipt 完工入库
↗ OutsourcingOrder 委外单（虚拟库位模型）    ↗ ProductionInspection 生产报检

ProductionPlan 生产计划

兼容按单生产(MTO)与按库存备货(MTS)双轨制排产

entity ProductionPlan {
  id: UUID PK
  doc_number: String // PP-2026-05-xxxxx
  plan_date: Date
  plan_type: PlanType
  status: PlanStatus
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

entity ProductionPlanItem {
  id: UUID PK
  plan_id: UUID FK→production_plans
  product_id: UUID FK→products
  planned_qty: Decimal(10,6)
  scheduled_start: Date
  scheduled_end: Date
  sales_order_id: UUID? FK→sales_orders (MTO)
  sales_order_item_id: UUID? // 具体行号
  bom_snapshot_id: UUID? // BOM版本快照
  routing_id: UUID? FK→labor_processes
  work_center_id: UUID? // 生产中心
  priority: i32
  status: PlanItemStatus
}

enum PlanType {
  MTO // 按单生产
  MTS // 按库存备货
  }
enum PlanStatus {
  DRAFT, CONFIRMED, IN_PROGRESS
  COMPLETED, CANCELLED
  }
enum PlanItemStatus {
  PLANNED, RELEASED, IN_PRODUCTION
  COMPLETED, CANCELLED
  }

WorkOrder 工单

将生产计划细化到批次和班组，是领料与执行的唯一依据

entity WorkOrder {
  id: UUID PK
  doc_number: String // WO-2026-05-xxxxx
  plan_item_id: UUID? FK→production_plan_items
  product_id: UUID FK→products
  bom_snapshot_id: UUID? // 冻结的BOM快照
  routing_id: UUID? FK→labor_processes
  planned_qty: Decimal(10,6)
  completed_qty: Decimal(10,6)
  scrap_qty: Decimal(10,6)
  scheduled_start: Date
  scheduled_end: Date
  actual_start: Timestamp?
  actual_end: Timestamp?
  status: WorkOrderStatus
  work_center_id: UUID? // 车间/产线
  team_id: UUID? // 班组
  sales_order_id: UUID? // MTO关联
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

enum WorkOrderStatus {
  DRAFT // 待计划
  PLANNED // 已计划
  RELEASED // 已下达
  IN_PRODUCTION // 生产中
  COMPLETED // 已完工
  CLOSED // 已关闭
  CANCELLED
  }

WorkOrderRouting 工单工序

基于工艺路线生成，记录每道工序的计划与实际执行情况

entity WorkOrderRouting {
  id: UUID PK
  work_order_id: UUID FK→work_orders
  step_no: i32 // 工序序号
  process_name: String // 贴片/插件/组装...
  work_center_id: UUID?
  standard_time: Decimal? // 标准工时(小时)
  standard_cost: Decimal? // 标准人工成本
  planned_qty: Decimal(10,6)
  completed_qty: Decimal(10,6)
  defect_qty: Decimal(10,6)
  status: RoutingStatus
  is_outsourced: bool // 是否委外工序
  is_inspection_point: bool // 是否报检点
}

enum RoutingStatus {
  PENDING, IN_PROGRESS, COMPLETED
  SKIPPED
  }

WorkReport 报工记录

工人实时提报工序完成进度与不良数，为计件工资和成本核算提供数据

entity WorkReport {
  id: UUID PK
  doc_number: String // WR-2026-05-xxxxx
  work_order_id: UUID FK→work_orders
  routing_id: UUID FK→work_order_routings
  report_date: Date
  shift: ShiftType
  worker_id: UUID FK→users
  completed_qty: Decimal(10,6)
  defect_qty: Decimal(10,6)
  defect_reason: String?
  work_hours: Decimal // 实际工时
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
}

enum ShiftType {
  DAY // 白班
  NIGHT // 夜班
  }

OutsourcingOrder 委外单

委外=虚拟生产库位，发外协=调拨，供应商报工=同模型

entity OutsourcingOrder {
  id: UUID PK
  doc_number: String // OO-2026-05-xxxxx
  work_order_id: UUID? FK→work_orders
  routing_id: UUID? FK→work_order_routings
  supplier_id: UUID FK→suppliers
  product_id: UUID
  outsourcing_type: OutsourcingType
  planned_qty: Decimal(10,6)
  completed_qty: Decimal(10,6)
  unit_price: Decimal(10,6)
  scheduled_date: Date?
  status: OutsourcingStatus
  virtual_warehouse_id: UUID // 虚拟库位
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

enum OutsourcingType {
  FULL // 产品全委外
  PROCESS // 工序委外
  MATERIAL // 材料委外
  REWORK // 委外返工
  }
enum OutsourcingStatus {
  DRAFT, SENT, IN_PRODUCTION
  DELIVERED, RECEIVED, CLOSED
  CONVERTED_TO_INTERNAL // 转自制
  CANCELLED
  }

ProductionInspection 生产报检

关键工序或完工后自动触发报检

entity ProductionInspection {
  id: UUID PK
  doc_number: String // PI-2026-05-xxxxx
  work_order_id: UUID
  routing_id: UUID?
  product_id: UUID
  inspection_type: InspectionType
  sample_qty: Decimal(10,6)
  qualified_qty: Decimal(10,6)
  unqualified_qty: Decimal(10,6)
  result: InspectionResult
  inspector_id: UUID
  inspection_date: Date
  disposition: String? // 处置意见
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
}

enum InspectionType {
  FIRST_ARTICLE // 首件检查
  IN_PROCESS // 巡检
  FINAL // 完工检
  }
enum InspectionResult {
  PASS, FAIL, CONDITIONAL
  }

ProductionReceipt 完工入库

检验合格后入库，触发倒冲和成本归集

entity ProductionReceipt {
  id: UUID PK
  doc_number: String // PR-2026-05-xxxxx
  work_order_id: UUID FK→work_orders
  product_id: UUID
  received_qty: Decimal(10,6)
  warehouse_id: UUID
  zone_id: UUID?
  bin_id: UUID?
  receipt_date: Date
  status: ReceiptStatus
  backflush_triggered: bool // 是否触发倒冲
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
}

enum ReceiptStatus {
  DRAFT, CONFIRMED, CANCELLED
  }

MES 模块 × 共享层交互汇总

DocumentSequence → PP/WO/WR/OO/PI/PR 编号
DocumentLink → 计划→工单→报工→入库 | 工单→委外单 | 工单→领料单
InventoryReservation → 工单下达时 HARD 预留物料 | 领料消耗预留 | 完工释放剩余
CostEntry → 报工记人工/制费(借:在制品/贷:人工费) | 完工记成品入库 | 倒冲差异记损耗
委外虚拟库位 → 发外协=WMS调拨到虚拟仓 | 外协收货=从虚拟仓调回 | 转自制=工单移回内部
