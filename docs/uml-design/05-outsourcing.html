Outsourcing Management (OM) — UML Class Diagram

核心业务流

OutsourcingOrder 委外单 ──SENT──→ OutsourcingTracking 节点追踪 ──DELIVERED──→ IQC 检验 ──RECEIVED──→ CLOSED
↗ 转自制 → WorkOrder（MES 内部工单）
架构方向：委外供应商 = WMS 虚拟生产库位，发外协 = WMS 调拨，收回 = WMS 调回

OutsourcingOrder 委外单

支持全委外/工序委外/材料委外/委外返工四种模式，复用 MES 已有实体

entity OutsourcingOrder {
  id: UUID PK
  doc_number: String // OO-2026-05-xxxxx
  work_order_id: UUID? FK→work_orders
  routing_id: UUID? FK→work_order_routings
  supplier_id: UUID FK→suppliers
  product_id: UUID
  outsourcing_type: OutsourcingType
  planned_qty: Decimal(10,6)
  completed_qty: Decimal(10,6)
  unit_price: Decimal(10,6)
  scheduled_date: Date?
  status: OutsourcingStatus
  virtual_warehouse_id: UUID // WMS 虚拟库位
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

enum OutsourcingType {
  FULL // 产品全委外
  PROCESS // 工序委外
  MATERIAL // 材料委外
  REWORK // 委外返工
  }
enum OutsourcingStatus {
  DRAFT, SENT, IN_PRODUCTION
  DELIVERED, RECEIVED, CLOSED
  CONVERTED_TO_INTERNAL // 转自制
  CANCELLED
  }

共享层交互：DocumentSequence 编号 | CostEntry 外协成本归集 | DocumentLink 关联工单/采购

OutsourcingMaterial 委外发料明细

追踪每笔委外单的物料发出/回收/在途状态，确保外包资产不流失

entity OutsourcingMaterial {
  id: UUID PK
  outsourcing_id: UUID FK→outsourcing_orders
  product_id: UUID FK→products（物料）
  planned_qty: Decimal(10,6) // 应发数量
  sent_qty: Decimal(10,6) // 已发数量
  returned_qty: Decimal(10,6) // 已收回数量
  unit_cost: Decimal(10,6) // 单位成本
}

计算公式：在途数量 = sent_qty - returned_qty
跨模块：发料/回收通过 WMS.InventoryTransferService 执行

OutsourcingTracking 委外追踪节点

快递式节点追踪，每个节点有 SLA 时间窗，超时自动升级预警

entity OutsourcingTracking {
  id: UUID PK
  outsourcing_id: UUID FK→outsourcing_orders
  node_type: TrackingNodeType
  tracked_at: DateTime // 实际时间
  planned_at: DateTime? // 计划时间（SLA）
  remark: String?
  operator_id: UUID
  created_at: Timestamp
}

enum TrackingNodeType {
  SEND_MATERIAL // 发料
  CARRIER_PICKUP // 承运商揽收
  SUPPLIER_RECEIVED // 供应商接收
  IN_PRODUCTION // 生产中
  SHIPPED // 发货
  IQC_INSPECTED // IQC检验
  WAREHOUSED // 入库
  }

节点流转链路

发料 → 承运揽收 → 供应商接收 → 生产中 → 发货 → IQC检验 → 入库
每节点可选配置 SLA 时间窗
超时 → 自动升级预警

共享层交互

DocumentSequence 编号
DocumentLink 关联委外单
CostEntry 外协收货→外协成本(借:在制品/贷:应付外协费)

Service 接口

interface OutsourcingOrderService {
  create(req, operator_id): OutsourcingOrder
  send(id, operator_id): OutsourcingOrder
  receive(id, received_qty, operator_id): OutsourcingOrder
  convert_to_internal(id, operator_id): WorkOrder
  find_by_id(id): OutsourcingOrder
  list(filter): Vec<OutsourcingOrder>
}
interface OutsourcingTrackingService {
  record_node(outsourcing_id, node_type, remark, operator_id): OutsourcingTracking
  list_by_outsourcing(outsourcing_id): Vec<OutsourcingTracking>
  check_overdue(): Vec<OutsourcingTracking>
}

OM 模块 × 跨模块交互汇总

MES.WorkOrderService → 委外单来源（工单→委外） | 转自制时创建内部工单
WMS.InventoryTransferService → 发外协=调拨到虚拟仓 | 收回=从虚拟仓调回
SRM.SupplierService → 委外供应商信息引用
QMS.InspectionResultService → 委外收货自动触发 IQC 检验
CostEntry → 外协收货时记外协成本(借:在制品/贷:应付外协费)
DocumentLink → 委外单↔工单 | 委外单↔追踪节点 | 委外单↔调拨单
