Quality Management (QMS) — UML Class Diagram

三道质量关口

IQC 来料检验 ← 紧咬 WMS 来料通知，不合格绝对无法采购入库
IPQC 过程检验 ← MES 首件检查 + 工序巡检，掐断批量报废苗头
FQC/OQC 终检/出货检 ← 出厂前批次抽检，不合格不予放行出库
MRB 不良评审 ← 报废/退货/降级/返工多方线上会签    RMA 客诉 ← 逆向触发内部质量追溯

InspectionSpecification 检验规格

统一注册表：定义每个产品在各检验阶段要检什么、公差多少、怎么抽样

entity InspectionSpecification {
  id: UUID PK
  doc_number: String // QS-2026-05-xxxxx
  product_id: UUID FK→products
  inspection_type: InspectionType
  check_items: JSONB // [{item, standard, tolerance, method}]
  sample_plan: JSONB // {level, aql, mode}
  status: SpecStatus
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

enum InspectionType {
  IQC // 来料检验
  IPQC // 过程检验（首件/巡检）
  FQC // 终检
  OQC // 出货检验
  }
enum SpecStatus {
  DRAFT, ACTIVE, INACTIVE
  }

设计理念：IQC/IPQC/FQC/OQC 仅是 inspection_type 枚举值，而非独立实体

InspectionResult 检验结果

实测记录：不合格自动阻断下游流转（数据层面硬门，非流程软约束）

entity InspectionResult {
  id: UUID PK
  doc_number: String // QR-2026-05-xxxxx
  spec_id: UUID FK→inspection_specifications
  source_type: InspectionSourceType
  source_id: UUID // 关联的来料通知/工单工序/发货单
  inspection_type: InspectionType
  batch_no: String
  sample_qty: Decimal(10,6)
  qualified_qty: Decimal(10,6)
  unqualified_qty: Decimal(10,6)
  result: InspectionResultType
  check_results: JSONB // [{item, measured, pass, remark}]
  inspector_id: UUID
  inspection_date: Date
  status: InspectionStatus
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
}

enum InspectionSourceType {
  ARRIVAL_NOTICE // 来料通知（→IQC）
  WORK_ORDER_ROUTING // 工单工序（→IPQC）
  SHIPPING_REQUEST // 发货申请（→OQC）
  OUTSOURCING_ORDER // 委外收货（→IQC）
  }
enum InspectionResultType {
  PASS, FAIL, CONDITIONAL
  }
enum InspectionStatus {
  PENDING, COMPLETED, DISPOSITIONED
  }

MRB 不良评审

组织技术、采购、质检进行多方线上会签，拒绝管理盲区

entity MRB {
  id: UUID PK
  doc_number: String // MRB-2026-05-xxxxx
  inspection_result_id: UUID FK→inspection_results
  product_id: UUID
  defect_description: String
  disposition: MRBDisposition
  responsible_party: ResponsibleParty
  cost_impact: Decimal(10,6)
  status: MRBStatus
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

enum MRBDisposition {
  SCRAP // 报废
  RETURN // 退货
  DEGRADE // 降级
  REWORK // 返工
  }
enum ResponsibleParty {
  INTERNAL, SUPPLIER, CUSTOMER
  }
enum MRBStatus {
  DRAFT, UNDER_REVIEW, APPROVED, COMPLETED
  }

RMA 客诉追溯

后端客诉逆向触发内部质量追溯，倒逼车间工艺或采购端质量改良

entity RMA {
  id: UUID PK
  doc_number: String // RMA-2026-05-xxxxx
  customer_id: UUID FK→customers
  sales_order_id: UUID? FK→sales_orders
  shipping_request_id: UUID? FK→shipping_requests
  product_id: UUID
  defect_description: String
  severity: Severity
  root_cause: String? // 根因分析
  corrective_action: String? // 纠正措施
  status: RMAStatus
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

enum Severity {
  MINOR // 轻微
  MAJOR // 重大
  CRITICAL // 致命
  }
enum RMAStatus {
  REPORTED, INVESTIGATING
  ACTION_TAKEN, CLOSED
  }

QMS 模块 × 跨模块交互汇总（质量关卡 = 状态门）

IQC → WMS 来料通知必须 IQC 通过才能入库（硬门：is_passed(ArrivalNotice, id) == true）
IPQC → MES 关键工序报检点自动触发（WorkOrderRouting.is_inspection_point == true）
OQC → Sales 发货申请必须 OQC 通过才能出库（硬门：is_passed(ShippingRequest, id) == true）
IQC → OM 委外收货自动触发 IQC 检验（硬门）
MRB → 不合格处置触发 CostEntry（返工/报废成本）+ WMS 库存调整
RMA → 客诉触发质量追溯，关联原工单/工序/来料批次 | DocumentLink 全链路
