三道质量关口

IQC 来料检验 ← 紧咬 WMS 来料通知，不合格绝对无法采购入库
IPQC 过程检验 ← MES 首件检查 + 工序巡检，掐断批量报废苗头
FQC/OQC 终检/出货检 ← 出厂前批次抽检，不合格不予放行出库
MRB 不良评审 ← 报废/退货/降级/返工多方线上会签（WorkflowEngine 编排）    RMA 客诉 ← 逆向触发内部质量追溯

Quality Gate 质量关卡 — check_gate() 三值判定

Passed 放行
存在活跃检验规格，且检验结果为 Pass/Conditional
Failed 阻断
存在活跃检验规格，但检验结果为 Fail 或尚在 Pending
NotRequired 免检
无活跃检验规格（产品未配置检验要求），允许放行

强制规则：调用方（如 WMS.confirm）禁止直接检查 bool，必须通过 check_gate() 获取明确状态后决策。Pending 状态归入 Failed（待检 = 不放行）。Conditional（让步接收）归入 Passed（业务语义：有条件放行 = 放行）。

事务模式 + 共享基础设施集成

TransactionMode 声明
InCallerTx（质量硬门）：
  check_gate(source_type, source_id, inspection_type) — 调用方事务内读取，保证硬门一致性
  WMS.confirm() → check_gate() 必须在同一事务内

IndependentTx（写操作）：
  create / update / record_result
  submit_for_review / execute_disposition
  record_root_cause / close

写操作内部自行管理事务边界。
跨模块副作用通过 DomainEventBus
异步解耦。
集成调用矩阵
DocumentSequence → create 时生成编号
StateMachine → 所有状态转换
AuditLog → 写操作同事务内
DomainEventBus → 操作完成后
  InspectionPassed → WMS 放行、MES 继续流转
  InspectionFailed → MRBHandler 自动创建 MRB
  MRBDispositioned → CostEntry + WMS 库存调整
DocumentLink → RMA 追溯链
CostEntry → MRB 报废/返工成本
Idempotency → record_result 防重
WorkflowEngine → MRB 多方会签

InspectionSourceType ↔ DocumentType 映射

QMS 内部使用 InspectionSourceType 语义明确，但与共享层交互时必须通过 DocumentType。实现层提供 from_document_type() 映射，上游单据演进不影响质量关卡。

InspectionSourceType::ArrivalNotice    ↔ DocumentType::ArrivalNotice
InspectionSourceType::WorkOrderRouting ↔ DocumentType::WorkOrderRouting
InspectionSourceType::ShippingRequest   ↔ DocumentType::ShippingRequest
InspectionSourceType::OutsourcingOrder  ↔ DocumentType::OutsourcingOrder

// 调用方传入 DocumentType，QMS 内部转换为 InspectionSourceType
// 新增单据类型时只需扩展映射函数，不影响已有检验逻辑

状态机流转声明（StateMachineService 驱动）

InspectionSpecification: Draft → Active → Inactive
InspectionResult: Pending → Completed → Dispositioned
  Guard: Pending → Completed 需 qualified_qty + unqualified_qty == sample_qty
MRB: Draft → UnderReview → Approved → Completed
  ↑ submit_for_review 触发 WorkflowEngine 审批实例
  ↑ execute_disposition 由 WorkflowHook.on_approved 回调触发
  Guard: UnderReview → Approved 需当前用户有审批权限 + 所有审批节点已完成
RMA: Reported → Investigating → ActionTaken → Closed
  ↑ record_root_cause 自动触发 Investigating → ActionTaken

工程约定（实现层约束）

JSONB 强类型 CheckItem / SamplePlan / CheckResult 派生 Serialize + Deserialize + Clone + Debug + PartialEq
错误链 使用 thiserror 定义 QmsError，通过 #[from] 兼容 DomainError::Internal(anyhow::Error)，gRPC interceptor 统一映射
