Financial Management (FMS) — UML Class Diagram

业财一体核心流转

销售回款 CashJournal(Inflow) → WriteOff（按出库单核销）→ 自动对应收账款
采购付款 CashJournal(Outflow) → WriteOff（按采购入库单核销）→ 自动对应付账款
费用报销 ExpenseReimbursement → 审批通过 → CashJournal(Outflow) → 成本归集
成本核算 CostEntry 统一账本 → 料/工/费完整归集 → 利润中心 P&L
架构方向：统一 cost_entries 分类账，每个产生成本的事件写入同一张表，双层记账

CashJournal 出纳日记账

管控所有现金流入流出：销售回款、采购付款、费用报销、工资支付，钱账实时对齐

entity CashJournal {
  id: UUID PK
  doc_number: String // CJ-2026-05-xxxxx
  journal_type: JournalType
  direction: CashDirection
  amount: Decimal(10,6)
  counterparty_type: CounterpartyType
  counterparty_id: UUID
  source_type: DocumentType
  source_id: UUID
  bank_account: String
  transaction_date: Date
  period: String // "2026-05"
  status: JournalStatus
  remark: String
  operator_id: UUID
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

enum JournalType {
  SALES_RECEIPT // 销售回款
  PURCHASE_PAYMENT // 采购付款
  EXPENSE // 费用报销
  PAYROLL // 工资支付
  OTHER // 其他
  }
enum CashDirection {
  INFLOW // 流入
  OUTFLOW // 流出
  }
enum CounterpartyType {
  CUSTOMER, SUPPLIER, EMPLOYEE, OTHER
  }
enum JournalStatus {
  DRAFT, CONFIRMED, CANCELLED
  }

CashJournalLine 日记账明细

双层记账：每笔日记账同时记借方和贷方，支持成本中心和利润中心归集

entity CashJournalLine {
  id: UUID PK
  journal_id: UUID FK→cash_journals
  account_code: String // 会计科目编码
  debit_amount: Decimal(10,6) // 借方金额
  credit_amount: Decimal(10,6) // 贷方金额
  cost_center: UUID? // 成本中心
  profit_center: UUID? // 利润中心
  remark: String
}

记账规则：每笔 CashJournal 至少两条明细行（一借一贷），借方总额 = 贷方总额

WriteOff 核销

销售回款按出库单核销，采购付款按入库单核销，确保每笔钱都有据可查

entity WriteOff {
  id: UUID PK
  write_off_type: WriteOffType
  cash_journal_id: UUID FK→cash_journals
  source_type: DocumentType
  source_id: UUID // 出库单/入库单
  amount: Decimal(10,6)
  write_off_date: Date
  operator_id: UUID
  created_at: Timestamp
}

enum WriteOffType {
  SALES_RECEIPT // 销售回款核销
  PURCHASE_PAYMENT // 采购付款核销
  }

核销规则：销售回款按出库单逐笔核销，确保每笔进账精准对齐到具体客户和发货批次

ExpenseReimbursement 费用报销

员工费用报销申请，审批通过后自动生成付款日记账

interface CashJournalService {
  create(req, operator_id): CashJournal
  confirm(id, operator_id): CashJournal
  find_by_id(id): CashJournal
  list(filter): Vec<CashJournal>
  get_balance(period): Decimal
}

interface WriteOffService {
  write_off(journal_id, source_type, source_id, amount, operator_id): WriteOff
  list_by_source(source_type, source_id): Vec<WriteOff>
  get_unreconciled_amount(source_type, source_id): Decimal
}
interface ExpenseReimbursementService {
  create(req, operator_id): ExpenseReimbursement
  submit(id, operator_id): ExpenseReimbursement
  approve(id, operator_id): ExpenseReimbursement
  pay(id, cash_journal_id, operator_id): ExpenseReimbursement
  find_by_id(id): ExpenseReimbursement
  list(filter): Vec<ExpenseReimbursement>
}

interface CostAccountingService {
  get_product_cost(product_id, period): ProductCostSummary
  get_work_order_cost(work_order_id): WorkOrderCostSummary
  get_profit_center_summary(period, profit_center): ProfitCenterSummary
  get_margin_analysis(order_id): MarginAnalysis
}

FMS 模块 × 跨模块交互汇总

Sales CRM → 销售回款 CashJournal(Inflow) + WriteOff 核销到 ShippingRequest
Purchase SRM → 采购付款 CashJournal(Outflow) + WriteOff 核销到 PurchaseOrder/ArrivalNotice
CostEntry 消费 → get_product_cost 汇总料/工/费 | get_work_order_cost 归集到工单
利润中心 → get_profit_center_summary 按部门归集 P&L | get_margin_analysis 报价-实际毛利差异
MES → 完工入库触发成品成本 | 报工触发人工成本 | 倒冲差异触发损耗成本
OM → 外协收货触发外协成本
QMS → 不良处置触发返工/报废成本
