总账模块 (GL) — 类图设计 v1

科目表 + 凭证三态 + 借贷平衡 + 期间锁定 + 明细账 + 试算平衡 + 期初余额 · 业财一体内核

General Ledger (GL) — UML Class Diagram v1

GL 核心流转

手工凭证 create_manual(Draft) → post(Posted, 借贷平衡校验) → cancel(Cancelled, 余额排除)
期间锁定 period.close(Open→Closed) → 该期 create_manual 失败(PeriodClosed)
试算平衡 trial_balance(period) → 总借=总贷 + 期末余额
明细账 general_ledger(account, from, to) → 分录流水 + running_balance
科目余额 get_account_balance(period, as_of_date) → opening_balance + Σ( posted 分录, 按方向定向累加)
架构方向：科目表(GlAccount) + 期间(GlPeriod) + 凭证(GlEntry/GlEntryLine) 三张核心表，支持期初余额、多币种、辅助核算

1. 科目表模型 (GlAccount)

entity GlAccount {
  id: i64 PK
  code: String UNIQUE -- 科目编码
  name: String
  account_type: AccountType -- 资产/负债/权益/收入/费用
  parent_id: i64? FK -- 父科目
  is_detail: BOOLEAN -- 是否末级(可用于凭证)
  balance_direction: BalanceDirection -- 借方/贷方
  company_id: i64
  reconcile: BOOLEAN -- 是否对账(应收/应付/税金)
  disabled: BOOLEAN -- 是否禁用(替代 status)
  opening_balance: DECIMAL(18,6) -- 期初余额
  currency: VARCHAR(10) -- 币种(默认 CNY)
  version: i32
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

enum AccountType { ASSET, LIABILITY, EQUITY, REVENUE, EXPENSE }
enum BalanceDirection { DEBIT, CREDIT }

设计要点：树形结构(parent_id)，只有末级科目(is_detail=TRUE)可用于凭证。balance_direction 决定余额计算公式：借方科目余额=期初+借-贷，贷方科目余额=期初+贷-借。reconcile 标记需要往来对账的科目。

2. 会计期间 (GlPeriod)

entity GlPeriod {
  id: i64 PK
  period: VARCHAR(7) UNIQUE -- "2026-06"
  start_date: DATE
  end_date: DATE
  status: PeriodStatus -- Open/Closed
  company_id: i64
  version: i32
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

enum PeriodStatus { OPEN, CLOSED }

期间锁定规则：close 要求期间无 draft 凭证(status=1)。已关闭期间不允许新建凭证(create_manual 自动 resolve_open 期间，失败则报 PeriodClosed)。

3. 凭证模型 (GlEntry/GlEntryLine)

entity GlEntry {
  id: i64 PK
  doc_number: String UNIQUE -- "GE-2026-06-xxxxx"
  period: VARCHAR(7) -- "2026-06"
  entry_date: DATE
  source_type: DocumentType -- 业务来源
  source_id: i64
  description: String
  voucher_type: VARCHAR(20) -- Journal Entry/Receipt/Payment/Contra/Opening
  is_opening: BOOLEAN -- 是否期初凭证
  status: EntryStatus -- 1draft/2posted/3cancelled
  total_debit: DECIMAL(18,6)
  total_credit: DECIMAL(18,6)
  operator_id: i64
  version: i32 -- 乐观锁
  created_at: Timestamp
  updated_at: Timestamp
  deleted_at: Timestamp?
}

entity GlEntryLine {
  id: i64 PK
  entry_id: i64 FK
  account_id: i64 FK → gl_accounts
  debit: DECIMAL(18,6)
  credit: DECIMAL(18,6)
  amount_currency: DECIMAL(18,6) -- 原币金额
  currency: VARCHAR(10) -- 原币币种
  exchange_rate: DECIMAL(18,6) -- 汇率
  cost_center: i64? -- 成本中心
  profit_center: i64? -- 利润中心
  project_id: i64? -- 项目
  memo: String
}

enum EntryStatus { DRAFT(1), POSTED(2), CANCELLED(3) }

凭证三态流转：Draft → Posted(post 校验借贷平衡 + 期间 open + 科目末级) → Cancelled(cancel 作废，余额查询自动排除 cancelled 凭证，无需红字反向分录)。

借贷平衡强制校验：post 时检查 total_debit == total_credit，不相等抛 DomainError::BusinessRule("UnbalancedEntry")。create 不校验，允许 draft 凭证暂不平衡。

4. GlEntryService trait

interface GlEntryService {
  create_manual(ctx, db, req): Result<i64> [InCallerTx]
  post(ctx, db, id): Result<()> [InCallerTx] -- 校验平衡+期间+末级，乐观锁更新 status=Posted
  cancel(ctx, db, id): Result<()> [InCallerTx] -- posted→cancelled，乐观锁，记审计
  get(ctx, db, id): Result<(GlEntry, Vec<GlEntryLine>)>
  list(ctx, db, filter: GlEntryFilter, page): PaginatedResult
  trial_balance(ctx, db, period): Result<TrialBalance> -- 只统计 posted 凭证，含 opening_balance
  general_ledger(ctx, db, account_id, from?, to?): Result<Vec<GlDetailRow>> -- 分录流水 + running_balance
  get_account_balance(ctx, db, account_id, period?, as_of_date?): Result<Decimal> -- 期初 + Σ(posted 分录，按方向定向)
}

// TransactionMode: InCallerTx (写操作与调用方同一事务)
// create/post/cancel 都有 AuditLog 同步记录

5. 核心算法说明

试算平衡 (trial_balance)

// 1. 按期间汇总所有科目的借/贷发生额
SELECT a.id, SUM(l.debit), SUM(l.credit)
FROM gl_accounts a
LEFT JOIN gl_entry_lines l ON l.account_id = a.id
LEFT JOIN gl_entries e ON e.id = l.entry_id
AND e.period = $1
AND e.status = 2 -- Posted
GROUP BY a.id

// 2. 计算期末余额
IF (balance_direction = DEBIT) {
  end_balance = opening_balance + period_debit - period_credit
} ELSE {
  end_balance = opening_balance + period_credit - period_debit
}

明细账 (general_ledger)

// 返回分录流水 + 累计余额
SELECT e.id, e.doc_number, e.entry_date, e.description,
     l.debit, l.credit,
     counterpart_account_id,
     running_balance
FROM gl_entries e
JOIN gl_entry_lines l ON l.entry_id = e.id
WHERE l.account_id = $1
AND e.status = 2 -- Posted
AND e.deleted_at IS NULL
ORDER BY e.entry_date ASC

// counterpart: 同凭证其他行第一行的 account_id
// running_balance: 按 balance_direction 累加

6. 业财一体衔接说明

Plan B (发票过账) → 销售发票/采购发票 post → 自动生成 GlEntry(借应收/贷收入、借库存/贷应付)
Plan C1 (收付款过账) → CashJournal.confirm → 同事务 post_from_source 一张 posted GlEntry：
  · SalesReceipt(收款)：借 default_bank / 贷 default_ar（应收减少）
  · PurchasePayment(付款)：借 default_ap / 贷 default_bank（应付+银行双减）
  · Expense：跳过（由 C2 统一过账）· Payroll/Other：跳过（无对方科目映射）
Plan C2 (报销付款过账) → ExpenseReimbursement.generate_payment_journal → 同事务 post_from_source：借 default_expense / 贷 default_bank
MES/WMS/QMS/OM → 完工入库/报工/不良处置 → 成本事件 → GlEntry(voucher_type=Opening/Contra)
自动分录 → 折旧/摊销/汇兑损益 → 定时任务 → GlEntry(source_type=System)
凭证来源 → source_type + source_id 记录原始单据，支持追溯和反查

7. 多币种 & 辅助核算字段

多币种支持 → GlEntryLine.amount_currency(原币) + currency + exchange_rate，本位币= debit/credit
辅助核算 → cost_center(成本中心) + profit_center(利润中心) + project_id(项目) 支持多维分析
期初凭证 → is_opening=TRUE 标识期初调整凭证，与 opening_balance 配合
凭证类型 → voucher_type 区分业务类型：Journal Entry(手工)/Receipt(收款)/Payment(付款)/Contra(冲销)/Opening(期初)

8. 数据一致性保障

乐观锁 → GlEntry.version + GlAccount.version 防并发冲突
软删除 → deleted_at 时间戳，查询自动过滤
审计 → AuditLog 记录关键操作(post/cancel)，status 变更有 Transition 日志
事务一致性 → create/post/cancel 都用 InCallerTx，与业务操作同事务提交/回滚
