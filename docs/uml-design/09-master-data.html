Master Data Module — UML Class Diagram

核心数据流

Category 分类（树形 + 物化路径） → Product 产品（M:N 多分类） → Bom 物料清单（草稿/发布） → BomNode 节点（树形结构）
↗ BomSnapshot 发布快照（不可变，工单引用）    ↗ PriceLogEntry 价格历史（支持时间点查询）    ↗ BomCostReport BOM 成本报告（支持 as_of_date）

Product 产品

核心基础实体，所有业务模块（库存/BOM/采购/销售/MES）的物料主数据。支持生命周期状态管理和外部编码

entity Product {
  product_id: i64 PK
  pdt_name: String // 产品名称
  product_code: String // 自动生成
  unit: String // 计量单位
  status: ProductStatus // Active/Inactive/Obsolete
  external_code: String? // 客户/供应商编码
  owner_department_id: i64? // 归属部门
  meta: JSONB → ProductMeta
}

enum ProductStatus {
  Active // 在用，出现在下拉框
  Inactive // 停用，不出现在下拉框
  Obsolete // 作废，仅供历史查询
}

struct ProductMeta {
  specification: String // 规格型号
  acquire_channel: String // 获取途径(采购/自制)
  old_code: String? // 旧编码(迁移兼容)
}

entity ProductCategory {
  product_id: i64 FK→products
  category_id: i64 FK→categories
  // UNIQUE(product_id, category_id)
}

trait ProductService {
  create(product, executor) → i64
    // 内部自动生成编码(x+时间戳)
  update(product_id, product, executor)
  delete(product_id, executor)
  find(product_id) → Option<Product>
  find_by_ids(ids) → Vec<Product>
  query(query) → (Vec, total)
  check_product_usage(id, query) → PaginatedResult
  update_status(id, status, executor)
}

Category 产品分类

产品分类树，支持无限层级。通过 ProductCategory 关联表实现产品多分类（一个产品可归属多个分类）。物化路径(path)优化子树查询性能

entity Category {
  category_id: i64 PK
  category_name: String // 分类名称
  parent_id: i64 // 父分类(0=顶级)
  path: String // 物化路径 如 "/1/5/12/"
  meta: JSONB → CategoryMeta
  created_at: DateTime
  updated_at: DateTime?
}

struct CategoryMeta {
  count: i64 // 关联产品数量
}

struct CategoryTree {
  category_id, category_name
  parent_id, path, meta
  children: Vec<CategoryTree>
}

trait CategoryService {
  create(req, executor) → i64
  update(category_id, req, executor)
  delete(category_id, executor)
  get(category_id) → Option<Category>
  list(query) → Vec<Category>
  get_tree(root_id?) → Vec<CategoryTree>
    // null=全树, id=子树
  move_to(category_id, new_parent_id,
    executor) // 级联更新path
  assign_products(category_id,
    product_ids, executor)
  remove_products(category_id,
    product_ids, executor)
}

BOM 物料清单

定义产品的物料构成。草稿/发布双状态，发布时生成不可变快照（BomSnapshot）。支持复制、物料替换、成本核算

entity Bom {
  bom_id: i64 PK
  bom_name: String
  create_at: DateTime
  update_at: DateTime?
  bom_detail: BomDetail // 从 bom_nodes 加载
  bom_category_id: i64? FK→bom_categories
  status: BomStatus
  published_at: DateTime?
  created_by: i64? FK→users
}

enum BomStatus {
  DRAFT // 仅创建者可见
  PUBLISHED // 全员可见
}

entity BomCategory {
  bom_category_id: i64 PK
  bom_category_name: String
  created_at: DateTime
}

BomNode BOM 节点

树形物料节点。parent_id=0 为根节点（成品），其余为子节点（原材料/半成品）。存储于独立的 bom_nodes 表。order 仅在 parent_id 相同的兄弟节点内有效，通过 swap_node_position 调整

entity BomNode {
  id: i64 PK
  bom_id: i64 FK→boms
  product_id: i64 FK→products
  product_code: String? // 冗余编码
  quantity: Decimal(10,6)
  parent_id: i64 // 0=根节点
  loss_rate: Decimal // 损耗率
  order: i32 // 兄弟节点内排序
  unit: String?
  remark: String?
  position: String? // 位置
  work_center: String? // 工作中心
  properties: String? // 自定义属性
}

PriceLogEntry 价格日志

每次价格变更自动追加记录，不可修改。支持多价格类型（采购价/销售价/标准成本）和按时间点查询历史价格

enum PriceType {
  Purchase // 采购价
  Sales // 销售价
  StandardCost // 标准成本
}

entity PriceLogEntry {
  log_id: i64 PK
  product_id: i64 FK→products
  price_type: PriceType
  new_price: Decimal(10,6)
  operator_id: i64?
  remark: String?
  created_at: DateTime
}

trait ProductPriceService {
  update_price(product_id, price_type,
    new_price, operator_id,
    remark, executor)
  query_price_history(query)
    → PaginatedResult
  get_current_price(product_id,
    price_type) → Decimal?
}

BomCostReport BOM 成本报告

汇总 BOM 的材料成本和人工成本。支持 as_of_date 按历史时间点核算，含价格缺失预警

struct BomCostReport {
  bom_id: i64
  bom_name: String
  product_code: String
  as_of_date: DateTime? // null=最新
  material_costs: Vec
  labor_costs: Vec
  warnings: Vec<String>
}

struct MaterialCostItem {
  node_id, product_id
  product_name, product_code
  quantity, unit_price?
}

struct LaborCostItem {
  id, name, unit_price
  quantity, sort_order, remark
}

BomService 关键操作

BOM 全生命周期管理：创建、编辑（乐观锁）、发布（生成快照+版本）、复制、替换、成本核算

// 并发控制
// bom.version 乐观锁
// UPDATE ... SET version = version + 1
// WHERE bom_id = $1 AND version = $2

// 发布流程（快照模式）
publish(bom_id, operator_id)
  Draft → Published
  // version 自增
  // 自动生成 BomSnapshot(bom_id, version, ...)
  // 发布事件: BomPublished
unpublish(bom_id, operator_id)
  Published → Draft
  // 不删除快照

// 物料替换（InCallerTx + FOR UPDATE）
substitute_product(
  old_product_id, new_product_id,
  bom_id?, overrides, caller_id)
  → (affected_boms, affected_nodes)

// 快照查询
get_snapshots(bom_id)
  → Vec<BomSnapshot> // 按 version DESC
get_snapshot_by_version(
  bom_id, version) → BomSnapshot

主数据模块 × 下游模块依赖

Product → 所有模块的物料标识 | Active/Inactive/Obsolete 状态控制可见性 | external_code 支持外部编码
Category → 产品分类筛选（物化路径 path 支持子树快速查询，通过 ProductCategory M:N 关联）
BOM → 乐观锁（version 字段） | MES 工单引用 BomSnapshot（不可变） | WMS 领料单物料来源
BomSnapshot → publish 时自动生成（version 自增），工单/MES 引用 snapshot_id 或 bom_id + version。unpublish 不删除快照
BomNode → get_leaf_nodes() 供出库使用 | loss_rate 用于成本差异计算 | order 仅在兄弟节点内有效
PriceLog → 多价格类型（Purchase/Sales/StandardCost） | FMS CostEntry 材料成本 | 支持 as_of_date 历史价格
权限 → PRODUCT / CATEGORY / BOM / PRICE / BOM_COST / BOM_LABOR_COST 6 种资源权限

共享基础设施集成点

DocumentSequenceService → ProductService.create() 内部自动生成编码（x+时间戳格式）
AuditLogService → 所有 create / update / delete / publish / unpublish 操作内调用 record()（同事务内，字段级 diff）
StateMachineService → BomStatus (Draft ↔ Published)、ProductStatus (Active → Inactive → Obsolete) 状态转换管理
DomainEventBus → 细粒度事件：BomPublished / BomUnpublished / BomNodeAdded / BomNodeUpdated / BomNodeDeleted / BomSubstituted / ProductStatusChanged

事务模式（TransactionMode）

InCallerTx — 调用者提供事务：substitute_product, add_node, update_node, delete_node, swap_node_position
RequiresNew — 方法自管事务：publish（生成快照+事件+审计）, unpublish, create, update, delete
None — 无事务：find, query, get_snapshots, get_bom_cost_report, get_leaf_nodes
