 # 制造企业 ERP 系统功能架构与业务蓝图分析报告（ABT 系统）

## 摘要
本报告针对企业当前运行的 **ABT ERP 系统** 进行深度模块解构与业务流分析。该系统是一个典型的**业财一体化、柔性化、全链路闭环**的数字化工厂解决方案。系统以**产品主数据**为基石，以**供-产-销-存**为主线，以**质量管理**为卡口，最终收敛于**财务控制中心**，支撑企业精细化运营。


---

## 第一章：基础数据源头 —— 产品管理系统（PDM）
产品管理是整个 ERP 的"数据心脏"，负责将研发/工程语言转化为系统可识别的标准化生产语言。

### 核心功能
* **工程BOM（E-BOM）：** 承接技术研发的原始物料清单，实现一物一码，支持**标签与二维码在线打印**，为全流程扫码溯源打下基础。
* **制造BOM（M-BOM）：** 转化为车间实际生产配方，考虑物料替代与损耗率，**直接联动采购算料与生产领料限额**。
* **工艺路线：** 规范产品加工的标准化标准工序（如：贴片 -> 插件 -> 组装 -> 灌胶 -> 质检 -> 包装），作为车间报工与财务核算工时工资的绝对依据。
* **工程变更（ECO）：** 管控 BOM 的版本更迭，作为避免前端乱改、后端做错货的"硬刹车"机制。

> **企业现状透视：** 数据显示企业对"半成品"（如贴片、焊线、注塑、PCBA等半成品）的层级划分极为精细，呈现出**多工序、多层级**的复合制造特点。

### 优化升级点
1. 配置化/参数化 BOM，支持通过参数自动生成不同配置版本。
2. BOM 多版本智能对比与影响分析（自动列出受影响工单、采购单、订单）。
3. 工艺路线增加标准工时、标准成本及关键质量特性（CTQ）标记。
4. 替代料分层管理（首选/备选/禁用）与动态损耗率。
5. ECO 变更自动推送与待办任务生成。

## 第二章：业务拉动龙头 —— 销售管理系统（CRM/OM）
销售管理是企业经营的源头，负责将外部客户需求转化为内部的生产动能。

### 核心功能
* **销售报价：** 结合产品 BOM 成本，规范前端销售的商务询报价行为，锁定订单初始利润。
* **订单评审与变更：** 组织技术、生产、财务进行多方线上会签，评估产能、交期与技术可行性，控制合同风险；变更单则全程留痕，杜绝口头通知。
* **发货申请：** 销售发货的指令枪，**直接流转至仓库触发实体出库流程**。
* **月对账单：** 月底一键导出客户发货明细，双方核对无误后**自动跨模块生成财务的应收账款**，打通业财融合第一步。

### 优化升级点
1. 智能报价引擎（实时基于最新 BOM 成本 + 工时计算预估毛利）。
2. 订单自动评审 + ATP/CTP 可用量与产能检查。
3. 订单变更影响模拟与预警。
4. 合同全生命周期可视化跟踪。
5. 客户信用额自动管控。
6. 月对账单一键生成与差异自动标记。

## 第三章：物资供应大闸 —— 采购管理系统（SRM）
采购管理负责控制企业的资金流出，确保生产不缺料、库存不积压。

### 核心功能
* **采购报价：** 登记各供应商的商品单价及**严格的报价有效期**，实现阳光比价。
* **采购订单：** 依据物料短缺或安全库存，向供应商正式下达购买指令。
* **对账付款闭环：** 通过"月对账单" -> "对账付款汇总表" -> "发票登记" -> "付款申请"的刚性流程，确保**每笔付出去的钱都有入库单和发票一一对应**。
* **零星请购/采购：** 为非生产物资开辟绿色通道，保障企业行政与日常后勤的敏捷运转。
* **供应商管理：** 维护供应商主档案（基本信息、资质证照、银行账户、联系人），支持供应商准入审核与分级分类，实现供应商全生命周期管控。

### 优化升级点
1. 供应商轻量报价门户（在线报价、自动比价）。
2. 采购框架协议管理（长期价格 + 阶梯折扣）。
3. MRP 驱动的采购需求自动生成与合并。
4. 供应商绩效自动评分（准交率、质量、价格）。
5. 三单匹配（入库+发票+付款）智能审核。
6. 采购成本异常预警。

## 第四章：物流与实体大本营 —— 库存管理系统（WMS）
库存管理是串联起供应商、车间、销售的物资中枢。

### 核心功能
* **来料通知与入库：** 供应商送货后先进入"来料通知"待检区（不计入可用库存），质检合格后流转为"采购入库"，真正增加原材料资产。
* **生产发/退/补料：** 严格依据工单配额发料。针对炸板、报废等异常提供"补料申请"以监控异常成本；完工余料通过"退料申请"回库。
* **系统倒冲：** 针对螺丝、胶水、贴片小电容等高频零碎散料，采用完工后系统自动按 BOM 反向扣减的倒冲机制，大幅简化仓管员的发料工作量。
* **高级仓储调度：** 支持库存调拨（多仓流转）、**形态转换**（大盘料剪切）、**库存锁定**（大客户预留锁库），具备极高的精细度。

### 优化升级点
1. 多策略库存计划（安全库存、最小最大库存、订货点等）。
2. 形态转换（大盘→小盘等）与成本自动分摊。
3. 批次/序列号高级管控 + 保质期 FEFO 出库。
4. ABC 分类 + 智能上架建议。
5. 循环盘点与差异自动处理。
6. 异常补料智能审批与成本归集。

## 第五章：车间制造引擎 —— 生产管理系统（MES）
生产管理是制造企业的核心。系统采用了 **自制流程 + 委外流程 + 管理报表** 的三维矩阵架构。

### 核心功能
* **双模式驱动排产：** 完美兼容 **按单生产（MTO）**（挂钩客户与销售订单号）与 **按库存备货（MTS）**（针对热销半成品提前平衡产能）的双轨制排产。
* **生产工单与派工：** 将生产计划细化到批次和车间班组，是领料与执行的唯一依据。
* **工序报工：** 工人实时提报每日各工序的完成进度与不良数，动态追踪生产生命周期（待计划 -> 已计划 -> 已申请 -> 已入库），同时为财务计件工资提供数据。
* **生产报检：** 关键工序或完工后自动触发报检，联动品质部门。

### 优化升级点
1. 轻量高级排产（APS）功能，支持瓶颈工序优先与拖期预警。
2. 工单智能拆分与合并。
3. 移动端扫码报工 + 不良原因代码记录。
4. 在制品 WIP 实时看板。
5. 生产异常在线提报与闭环处理。
6. 计件/计时工资基于报工自动计算。

## 第六章：打破围墙的产能 —— 委外管理系统（OM）
委外管理将企业外部供应商的产能完全纳入了数字化管控体系。

### 核心功能
* **多模式委外覆盖：** 支持**产品全委外**（外包加工）、**工序委外**（个别工序外协）、**材料委外**（改制加工）以及异常情况下的**委外返工**。
* **材料在途查询：** 实时监控发放在各外协厂手里的原材料存量，确保外包资产不流失。
* **极具弹性的控制链：** 系统支持【合并委外单】以压低外协单价；更支持**【转自制】**高级功能，当外协厂产能爆仓时，可一键将委外单切回内部车间生产，保障交付。

### 优化升级点
1. 委外供应商协同门户（在线接收、报工、上传检验报告）。
2. 委外材料全链路在途可视化。
3. 委外价格智能比价与框架协议。
4. 一键转自制或转其他供应商。
5. 委外成本自动归集。
6. 委外来料自动触发 IQC。

## 第七章：品质御林军 —— 质量管理系统（QMS）
质量管理是保障企业品牌与降低隐性成本的防火墙。

### 核心功能
* **三道质量关口：**
  1. **来料检验（IQC）：** 紧咬仓库"来料通知"，单单相连，不合格绝对无法办理采购入库。
  2. **过程检验（IPQC）：** 车间首件检查与工序巡检，及时掐断批量报废苗头。
  3. **产品终检（FQC/OQC）：** 出厂前最终批次抽检，不合格不予放行出库。
* **不良评审（MRB）：** 针对不良品组织技术、采购、质检进行多方线上会签（报废/退货/降级/返工），拒绝管理盲区。
* **客户投诉（RMA）：** 后端客诉逆向触发内部质量追溯，倒逼车间工艺或采购端进行质量改良。

### 优化升级点
1. 质量检验计划与 AQL 抽样标准自动生成。
2. SPC 统计过程控制（关键参数控制图 + 自动报警）。
3. 不良品 MRB 智能路由审批。
4. 批次/序列号全流程质量追溯。
5. RMA 逆向触发 CAPA 改进任务。
6. 质量 KPI 实时看板。

## 第八章：终极数字化晴雨表 —— 财务管理系统（FMS）
该财务模块是一个典型的"非孤立记账系统"，处理着高频的真实流水，是业财一体化的终极体现。

### 核心功能
* **出纳与日记账：** 严格管控销售回款、采购付款、费用报销及工资支付，钱账实时对齐。
* **高标准业务核销：** 系统内的销售回款统一**【按销售出库单】**进行依据核销。这意味着每一笔进账都能精准对齐到具体的客户和具体的发货批次，完全杜绝了呆账和糊涂账。
* **精细化成本与多中心核销：**
  * 存货管理与成本管理深度配合，精准核算每个产品的**料、工、费**，算出真实毛利率。
  * 收付款时自动按部门进行**利润中心归集**，让高层能够随时掌握各独立业务单元的盈利看板。

### 优化升级点
1. 实时成本核算（料、工、费完整归集）。
2. 报价-预估毛利-实际毛利差异分析。
3. 多维度利润中心自动归集。
4. 业务单据自动生成凭证。
5. 发票智能匹配与税务报表导出。

## 第十章：架构级设计方向（构思评审结论）

> 以下 6 个方向来自对当前代码库和本蓝图的系统性构思评审，经过 47 个原始想法的生成、交叉验证和对抗性过滤后保留。这些方向跨模块生效，应在各模块实现前先确定。

### 1. 可追溯性基础设施：统一文档编号 + 跨模块文档关联图谱

**问题：** 九大模块各自编号，事后对齐几乎不可能；模块间的单据关系（报价→订单→采购单→来料→检验→领料→工单→发货→发票）缺少统一的关联机制。

**方案：**
- 建立内核级文档编号服务（类型前缀 + 日期 + 序列号，如 `PO-2026-05-00142`），所有模块消费同一服务
- 建立 `document_links` 有向图表（source_type, source_id, target_type, target_id, link_type），任何模块间文档关系通过一次图遍历查询
- 受益方：统一待办中心、BI 报表、财务审计、ISO 9001 / IATF 16949 可追溯性合规

**复杂度：** 中等 | **信心：** 92% | **时机：** 必须在 CRM/SRM/MES/QMS/FMS 开发前确定

### 2. PDM 根基：BOM 展开引擎 + 版本快照

**现状：** 当前 BOM 已采用关系型邻接表模型（`bom_nodes` 表，`parent_id` 自引用），支持多级树结构、Draft/Published 状态、成本报告。

**缺失：**
- **BOM 展开引擎：** 输入产品ID + 数量 → 扁平化物料需求（含替代料、损耗率折叠）+ 工艺路线步骤 + 预估成本。一个引擎服务五个下游消费者：采购需求、领料、报工、质检排程、成本核算
- **版本快照：** 工单创建时冻结 BOM 快照，避免后续 BOM 变更影响在制工单
- **替代料建模：** 当前缺少替代料（首选/备选/禁用）的数据结构
- **变更影响分析：** ECO 变更时自动计算影响半径（受影响的工单、采购单、订单）

**复杂度：** 高 | **信心：** 85% | **时机：** MES/SRM 开发前必须完成

### 3. 库存预留层：硬/软分配协调机制

**问题：** "我们有 1000 个单位"和"200 个可用"是本质区别。当前系统只有现有量，没有可用量（ATP）。

**方案：**
- 引入 `inventory_reservations` 服务层，预留类型：工单硬预留、销售订单软预留（带优先级）、安全库存永久预留
- `可用量` = `现有量` − `已预留量`，预留支持 TTL 自动过期
- 采购建议 = 需求 − 现有量 − 已预留，这个公式没有预留层就无法计算

**受益方：** SRM 采购建议、MES 排产物料检查、CRM 发货可用量检查、WMS 库存优化

**复杂度：** 中等 | **信心：** 90% | **时机：** SRM/MES 开发前必须完成

### 4. 委外统一：虚拟生产库位 + 节点追踪

**问题：** 传统方案将委外建模为独立模块，导致独立表结构、独立物料追踪、独立报工——全部重复 WMS 和 MES 已有能力。

**方案：**
- 委外供应商 = WMS 中的虚拟生产库位。发外协 = 库间调拨，供应商报工 = 同模型不同 actor，"转自制" = 工单移回内部库位
- 快递式节点追踪：定义检查点（发料→承运→供应商接收→生产→发货→IQC→入库），每点有 SLA 时间窗，超时自动升级
- 现有库位占用校验、入库、出库流程完全不变

**复杂度：** 中等 | **信心：** 83% | **时机：** OM 模块设计时确定

### 5. 质量关卡架构：状态门覆盖层 + 统一检验注册表

**问题：** 将质量作为独立模块意味着 4+ 个双向集成接口，每个都有一致性缺口。传统做法容易被绕过。

**方案：**
- 质量不是独立模块，而是施加在实体状态转换上的**强制前置条件**
- 统一 `inspection_specifications` 注册表（检查什么、公差、抽样方案）+ `inspection_results` 表（实测值、合格/不合格、处置）
- IQC/IPQC/FQC/OQC 仅是 `inspection_type` 枚举值，而非独立实体
- 不合格自动阻断下游流转（数据层面硬门，非流程规范软约束）

**复杂度：** 中高 | **信心：** 88% | **时机：** QMS 模块设计时确定

### 6. 实时成本累积账本：双层记账从第一天开始

**问题：** 传统 ERP 陷阱——运营模块先建、财务后加，事后对账痛苦。

**方案：**
- 统一 `cost_entries` 分类账：每个产生成本的事件（物料入库→材料成本，工序完工→人工/制费，检验不良→返工/报废成本，外协收货→外协服务成本）写入同一张表
- 统一 schema：entity_type, entity_id, cost_type, amount, cost_center, profit_center, timestamp
- 双层记账原则：每个操作同时记借方和贷方
- FMS 报表、利润中心 P&L、毛利分析全部从这一张表查询
- 新增成本来源（如能源计量）只需插入行，无需修改 FMS 逻辑

**复杂度：** 高 | **信心：** 82% | **时机：** 第一个产生成本的模块（SRM 或 MES）开发前确定

### 7. 仓储精细化：三级库位模型 + 策略引擎 + 用量差异闭环

**问题：** 当前 WMS 有调拨、形态转换、锁库能力，但停留在"仓库级"粒度。缺少库位管理意味着无法指引仓管员"放哪里、从哪拿"，收货/拣货效率完全依赖人工经验。倒冲机制虽有，但缺少理论用量与实际用量的差异闭环，损耗率校正靠人工拍脑袋。

**方案：**

**三级库位模型：**
- 仓库（Warehouse）→ 库区（Zone，如原料区/成品区/待检区/退货区）→ 储位（Bin，物理最小单元，含行列层坐标）
- 每个储位维护属性：容量上限、允许物料类型、温控要求、状态（空闲/占用/锁定）
- 入库/出库/调拨全部精确到储位，库存查询可下钻到"某物料在哪个货架的哪一层"

**上架策略引擎（配置化）：**
- 收货入库时系统自动推荐储位，策略可配置：
  - **同物料合并**：优先放入已有该物料的储位（减少散布）
  - **就近入库**：放在离收货区最近的空储位
  - **指定储位**：特定物料绑定固定储位（如大件/危化品）
  - **空储位优先**：自动选择利用率最低的储位
- 策略可按物料类型、仓库区域分别配置，运行时自动匹配

**拣货策略引擎（配置化）：**
- 拣货出库时系统自动分配拣货任务与路径，策略可配置：
  - **FIFO（先进先出）**：优先拣最早入库的批次（默认策略）
  - **FEFO（先到期先出）**：优先拣保质期最近的批次（适用胶水/化学品）
  - **最短路径**：按储位物理位置规划最优拣货路径，减少走动距离
  - **整托优先**：当需求量达到整托时优先拣整托，减少拆零

**波次发货管理：**
- 多个销售订单合并为一个"发货波次"，统一生成拣货任务
- 流程：波次创建 → 拣货任务分配（按人/区域）→ 拣货确认 → 分拣复核 → 打包 → 发货确认
- 支持部分拣货、缺货登记、异常挂起

**循环盘点增强：**
- 基于 ABC 分类的差异盘点频率：A 类物料月盘、B 类季盘、C 类年盘
- **盲盘机制**：盘点任务只显示储位号，不显示系统库存量，避免"照抄"
- 盘点差异自动生成调整单，经审批后更新库存，同时记录差异原因代码
- 盘点计划可按库区/储位范围灵活调度，不影响正常出入库

**物料差异分析闭环：**
- 每次生产领料/倒冲完成后，自动对比：BOM 理论用量 vs 实际领料/倒冲量
- 差异率 =（实际用量 − 理论用量）/ 理论用量
- 差异超过阈值（如 ±5%）自动触发异常报告，推送生产主管确认
- 累计差异数据反哺 BOM 损耗率校正：连续 N 次超耗 → 建议调高 BOM 损耗率
- 成本模块联动：差异成本自动归集到工单成本，确保实际成本核算准确

**数据模型核心：**
```
warehouses → zones → bins
  ├─ bin_attributes (容量/类型/温控)
  ├─ stock_ledger (物料 × 批次 × 储位 的实时库存)
  └─ putaway_strategies / pick_strategies (配置化策略表)
```

**复杂度：** 中高 | **信心：** 88% | **时机：** WMS 模块重构时同步实施，第一期启动

### 8. 集成架构层：API 网关 + 事件总线 + 主数据治理

**问题：** 当前九大模块之间的集成依赖"模块间直接调用"，耦合度高。缺少统一的对外接口策略，无法高效对接外部系统（快递追踪、银行支付接口、税控开票、客户/供应商门户）。主数据（物料编码、客户、供应商）缺少统一治理入口，各模块各自维护，编码冲突和重复数据风险随业务增长急剧上升。

**方案：**

**API 网关层：**
- 所有模块对外暴露统一的 RESTful API，通过网关统一管理鉴权、限流、日志
- 外部系统对接标准化：快递物流追踪、银行回款通知、税控开票接口、电商平台订单同步，全部通过网关适配器接入
- API 版本管理（v1/v2 并存），确保接口升级不破坏已有集成
- Webhook 机制：关键业务事件（订单状态变更、发货完成、付款到账）主动推送给订阅方

**事件驱动架构（Event Bus）：**
- 引入轻量级事件总线（如 Redis Streams / NATS），核心业务动作发布领域事件：
  - `order.confirmed` → 触发排产计划生成
  - `goods.received` → 触发 IQC 检验任务
  - `inspection.failed` → 触发 MRB 流程 + 阻断入库
  - `production.completed` → 触发成本归集 + 库存更新
- 模块间通过事件解耦，新增功能只需订阅已有事件，无需修改上游模块
- 事件持久化 + 重试机制，确保关键业务不丢消息

**主数据治理平台（MDM）：**
- 统一编码规则引擎：物料编码、客户编码、供应商编码全部由中央服务生成，禁止各模块自行编号
- 唯一性校验：新增客户/供应商时自动查重（名称、税号、银行账号多维度匹配）
- 数据质量评分：定期扫描主数据完整度（缺失字段比例、过期数据、孤立数据），输出治理报告
- 变更审批流：主数据修改走统一审批，变更历史全程可追溯

**低代码流程引擎：**
- 审批流可视化配置：业务人员可自行拖拽配置审批节点、条件分支、超时自动升级
- 减少定制开发量：新业务流程（如"新增一类费用报销流程"）无需写代码
- 与统一待办中心联动，审批任务自动推送

**集成架构总览：**
```
外部系统 (快递/银行/税控/电商)
        ↕
    API 网关 (鉴权/限流/日志/版本)
        ↕
    事件总线 (发布/订阅/持久化/重试)
        ↕
  ┌─────┼─────┼─────┼─────┐
  CRM  SRM  MES  QMS  FMS  ...
  └─────┼─────┼─────┼─────┘
        ↕
    主数据治理 (编码/查重/质量/变更)
        ↕
    流程引擎 (审批流/待办/超时升级)
```

**复杂度：** 高 | **信心：** 80% | **时机：** 第零期确定 API 网关与事件总线规范，第一期与首个模块同步实施

## 总结：ABT 系统的核心价值与升级蓝图

ABT ERP 系统通过**业财一体化**与**全链路闭环**设计，已构建起坚实的数字化基础。优化升级后，系统将进一步实现：

1. **管理透明化：** 老板和管理层可以随时通过系统，看清任何一张订单停留在哪个车间、哪家外协厂，或者哪笔款项尚未回笼。
2. **损失最小化：** 通过限额领料、比价有效期、质量三道关口、BOM变更控制，把企业的物料损耗和经营风险降到最低。
3. **决策数据化：** 沉淀下来的流水和库存数据，为企业后续进行精准的供应商筛选、客户贡献度分析、车间产能优化提供了绝对可靠的大数据资产。

### 实施优先级建议（三期推进）

**第零期（架构先行）：** 统一文档编号服务 + 文档关联图谱 + 成本累积账本 + 库存预留层 + API 网关与事件总线规范 + 主数据治理编码规则
**第一期（1-3个月）**：PDM BOM展开引擎 + 版本快照 + 销售智能报价 + 三级库位模型 + 上架/拣货策略引擎 + 循环盘点增强
**第二期（3-6个月）**：MES 排产 + 委外虚拟库位 + 质量关卡架构 + 物料差异分析闭环 + 波次发货管理 + 低代码流程引擎
**第三期（6-9个月）**：全系统移动端 + 供应商协同 + 高级分析功能 + 外部系统集成适配器（快递/银行/税控）
