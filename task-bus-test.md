# Role: 资深 ERP 测试架构师 & AI 自动化测试专家

# Background:
我们的一款**生产型 ERP 系统**的底层框架已经开发完成。现在需要进行一次全面的 **E2E (端到端) 全链路业务流程测试**。
该系统的核心业务主线是 **“从销售报价到财务结算 (Quote-to-Cash)”**。
测试执行手段将采用前沿的 **`agent-browser` (基于 AI Agent 驱动的浏览器自动化)** 技术，模拟不同角色的真实用户操作。

# Task:
请为我设计一份深度、全面、可落地的《生产型 ERP 全链路业务流程自动化测试方案》。

# Requirements (请严格按以下维度展开设计):

## 1. 生产型 ERP 全链路业务节点拆解 (Quote-to-Cash)
请不要局限于简单的销售，必须体现 **“生产制造型”** 企业的业务深度。请梳理从“销售报价”到“财务结算”的完整主干流程，必须包含但不限于以下环节的联动：
- **销售端**：报价、合同、销售订单。
- **计划与生产端**：BOM 解析、MRP 运算、生产工单下达、车间接收与报工。
- **供应链端**：采购申请、采购订单、收货入库、生产领料、成品入库、销售发货。
- **财务端**：应收/应付账款生成、成本核算、开票、最终财务结算。
*(请以流程节点列表的形式呈现，明确每个节点的输入、输出及触发的下游动作)*

## 2. 跨模块联动、审批流与事件通知机制
ERP 的核心在于控制。请针对上述流程，设计“事件与审批矩阵”：
- **审批流设计**：哪些关键节点（如特价审批、超额采购审批、生产报废审批）需要触发工作流？不同角色的审批动作如何流转？
- **联动与通知机制**：节点状态变更时，系统应触发哪些站内信、邮件、钉钉/企微通知？（例如：生产完工通知仓库质检，发货通知财务开票）。请用表格形式输出。

## 3. 基于 `agent-browser` 的多角色自动化测试策略 (核心重点)
请详细说明如何利用 `agent-browser` 来执行上述全链路测试：
- **Agent 角色定义**：如何为不同的 Agent 赋予不同的系统角色（如销售专员、生产主管、仓库管理员、财务出纳）？
- **上下文与状态传递（接力测试）**：Agent A (销售) 创建订单后，Agent B (计划员) 如何通过 `agent-browser` 识别并获取该订单进行 MRP 运算？请说明测试数据的流转与断言机制。
- **异常与逆向操作模拟**：如何让 Agent 模拟“审批驳回”、“库存不足强行发货”、“生产报废”等异常场景，并验证系统的容错与数据回滚机制？

## 4. 测试数据准备与环境隔离
- 生产型 ERP 依赖大量基础数据（如物料主数据、BOM、工艺路线、客户/供应商档案）。请说明如何为 `agent-browser` 批量构造和注入这些高仿真测试数据。

## 5. 质量评估与准出标准
- 针对全链路 E2E 测试，定义明确的 Pass/Fail 标准（不仅是页面 UI 不报错，更要验证底层数据表的一致性，如业财一体化下的总账与明细账是否平）。
